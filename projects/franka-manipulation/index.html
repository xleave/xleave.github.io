Franka 机械臂 VLA 闭环操控与安全监管系统

基于视觉-语言-动作 (VLA) 模型的 Franka 机械臂闭环操控平台，集成了端侧细粒度微调、DROID 风格数据收集流水线及多级安全屏障。

GitHub 仓库

项目详情

项目背景

在机器人通用操控领域，大参数量的视觉-语言-动作 (VLA) 模型提供了强大的通用语义理解力，但在真机部署时面临动作高维控制不一致和不可靠的执行风险。本项目构建了一套适用于 Franka Emika 平台的高保真闭环执行与安全网关系统，提供端到端推理、真机轨迹收集、可控模型微调以及物理级防碰撞机制。

核心技术架构

1. VLA 推理服务与多模态任务理解

  • 混合推理架构：构建本地 VLA Server 操控后台，通过 Qwen2-VL 进行场景感知分析与多步任务规划，结合 GroundingDINO 提取三维物理坐标，在存在歧义的指令下支持 HRI 人机交互节点自适应对话澄清。
  • 动作控制接口：制定并适配统一的 7D 动作控制向量接口 [dx, dy, dz, droll, dpitch, dyaw, gripper]，通过自适应转换器打通了 MuJoCo 仿真、数据收集手柄、真机直连及微调数据的统一语义。

2. DROID-Style 轨迹数据流水线

  • 数据对齐与清洗：实现类似 DROID 项目的真机轨迹收集与标准化管理，统一对齐多视角相机帧、Franka 关节状态（Joint States）、末端力矩、自然语言任务描述及元数据。
  • 微调适配器：构建轻量级适配器用于本地数据归一化和去噪，生成项目级 dataset_statistics.json，为 OpenPI 等具身智能大模型在特定任务上的 LoRA 微调提供标准数据对齐规范。

3. 多层级安全监管网关 (Execution Gateway)

  • 安全沙盒机制：设计 HRI -- robot_gateway -- policy_sidecar -- Execution Bridge (RT Supervisor) 的四层执行隔离网关，限制 VLA 的连续直接执行权，转换为局部轨迹评估。
  • 物理围栏拦截：实现短期动作租约机制 ExecutionLease，在机械臂出现轨迹奇异性、逆运动学碰撞或置信度突降时，由物理网关直接拦截并阻塞，提供一键物理急停（E-Stop）和人工介入接管。
