VLA 模型架构演进：从 RT-2 到 π0 的设计范式变迁

#VLA#RT-2#π0#具身智能#大模型

视觉-语言-动作模型（Vision-Language-Action Models, VLA）是具身智能领域的核心研究方向。VLA 模型以视觉观测和语言指令为输入，直接输出机械臂的底层控制动作，实现端到端的感知-决策闭环。

从 2023 年 Google DeepMind 的 RT-2 到 2024 年 Physical Intelligence 的 π0，VLA 模型经历了数代架构迭代。本文梳理这一演进脉络，分析各代模型的设计范式与核心取舍。

1. RT-2：视觉-语言模型直接输出动作 Token

1.1 核心思想

RT-2（Robotics Transformer 2）的核心洞察是：既然视觉-语言模型（VLM）已经能理解图像并生成语言 Token，那么机器人动作也可以被表示为 Token 序列，由同一个模型生成。

RT-2 将连续动作空间离散化为 256 个 bin，每个动作维度（如 x/y/z 平移、旋转、夹爪开合）映射到一个整数 Token。模型的输出从”语言文本”变为”动作 Token 序列”，训练和推理流程与标准 VLM 保持一致。

1.2 架构

RT-2 基于 PaLI-X 或 PaLM-E 等预训练 VLM：

输入: [图像 Token] + [语言指令 Token]
     ↓
预训练 VLM Backbone（冻结或微调）
     ↓
输出: [动作 Token 1] [动作 Token 2] ... [动作 Token 7]
      (x, y, z, rx, ry, rz, gripper)

1.3 优势与局限

优势局限
继承了大规模 VLM 的视觉-语言理解能力动作离散化损失精度
训练流程统一，无需额外动作解码器自回归逐 Token 生成动作，推理延迟高
互联网规模预训练提供泛化先验模型参数量巨大（5B-55B），部署成本高

2. Octo：灵活的通用机器人策略基础模型

2.1 核心思想

Octo（2024, UC Berkeley）的目标是构建一个通用的机器人策略基础模型，支持不同的机器人形态（Robot Morphology）、不同的传感器配置和不同的动作空间，通过少量数据微调即可适配新任务。

2.2 架构设计

Octo 采用 Transformer Backbone，但在输入和输出上做了模块化设计：

  • 输入端：使用 Tokenizer 统一处理来自不同传感器（单臂/双臂、不同摄像头数量）的观测数据，并支持语言指令和目标图像两种任务规范（Task Specification）。
  • 输出端：使用可替换的动作头（Action Head），支持离散化 Token、连续均值回归以及 Diffusion Head 等多种动作解码方式。
观测 Token + 任务描述 Token
        ↓
  Transformer Backbone
        ↓
  Readout Token → Action Head（可替换）
        ↓
  连续动作序列

2.3 关键创新

  • 模块化 Action Head：将动作解码与主干网络解耦，不同的 Action Head 适配不同的动作空间需求。
  • Open X-Embodiment 数据集：使用来自 22 种机器人的 80 万条轨迹进行预训练，是当时最大的机器人操作数据集。
  • 高效微调：冻结 Backbone 参数，仅微调 Action Head 和少量适配层，即可快速适配新机器人。

3. OpenVLA：开源 VLA 的工程化实践

3.1 核心思想

OpenVLA（2024, Stanford/UC Berkeley）在 RT-2 的”VLM 输出动作 Token”范式基础上，使用开源 VLM（Prismatic-7B, 基于 LLaMA 2）替代闭源的 PaLI-X，构建了一个完全开源、可复现的 VLA 基线。

3.2 与 RT-2 的差异

设计维度RT-2OpenVLA
基础 VLMPaLI-X / PaLM-E（闭源）Prismatic-7B（开源）
视觉编码器ViTSigLIP + DinoV2（双编码器融合）
动作表示256 bin 离散化256 bin 离散化
训练数据RT 内部数据集Open X-Embodiment
可复现性不可复现完全开源

3.3 工程贡献

OpenVLA 的主要价值在于工程化：提供了完整的训练代码、数据处理管线、LoRA 微调方案以及部署脚本，使社区研究者能以可控成本进行 VLA 研究。

4. π0：Flow Matching 范式的突破

4.1 核心思想

π0（2024, Physical Intelligence）提出了一个关键的范式转变：放弃将动作离散化为 Token，改用 Flow Matching 直接在连续动作空间中生成动作序列。

这一转变的动机是：

  • 动作的离散化（binning）天然引入精度损失，且 bin 数量与维度数呈指数增长。
  • 自回归 Token 生成是顺序的，延迟随动作维度线性增长。
  • 精密操作任务对动作精度要求很高，离散化难以满足。

4.2 架构

π0 的架构由三部分组成：

  1. 预训练 VLM Backbone（PaliGemma）：处理视觉和语言输入，提取多模态特征。
  2. 动作专家（Action Expert）：一组专门处理动作 Token 的 Transformer 层，与 VLM 层通过交叉注意力交互。
  3. Flow Matching 动作头：以噪声动作为输入，通过去噪过程生成高精度连续动作序列。
[图像 Token] + [语言 Token]  →  VLM Backbone
                                    ↓ 交叉注意力
[噪声动作 Token]  →  Action Expert  →  Flow Matching Head
                                    ↓
                           连续动作序列 (Action Chunk)

4.3 Flow Matching vs. 离散化

维度离散化 Token（RT-2/OpenVLA）Flow Matching（π0）
动作精度受 bin 数量限制连续空间，理论无损
生成方式自回归，逐 Token并行去噪，固定步数
推理延迟随维度增长与维度无关（固定去噪步数）
多模态分布单模态（argmax）天然支持多模态分布

4.4 π0.5：思维链增强

π0.5 在 π0 基础上引入了高级语言推理能力：模型先以自回归方式生成”思维链”（Chain-of-Thought），描述子目标和执行计划，然后基于这些中间推理结果通过 Flow Matching 生成动作。这使得模型能够处理需要多步规划的复杂长时域任务。

5. 架构演进的核心趋势

5.1 动作表示：从离散到连续

RT-2 (离散 Token)  →  Octo (混合 Head)  →  π0 (Flow Matching 连续生成)

早期 VLA 模型受限于 VLM 的自回归范式，将动作强制离散化。随着对精密操作需求的提升，Flow Matching / Diffusion 等连续生成方法逐渐成为主流。

5.2 视觉编码：从单编码器到融合编码

RT-2 (ViT)  →  OpenVLA (SigLIP + DinoV2)  →  π0 (PaliGemma SigLIP)

双编码器融合（语义编码器 + 空间编码器）已被证明能同时捕获高级语义和精细空间信息。

5.3 训练数据：从单一到跨机器人

RT-2 (Google 内部)  →  Octo/OpenVLA (Open X-Embodiment)  →  π0 (大规模多源数据)

跨机器人形态的大规模数据集使 VLA 模型具备了更强的泛化能力。

5.4 模型规模与效率的权衡

模型参数量推理延迟特征
RT-25B-55B高（大模型 + 自回归）
Octo~93M低（轻量 Backbone）
OpenVLA~7B中（LLaMA 级别）
π0~3B中（PaliGemma + Flow Matching）

6. 未来方向

  • 统一世界模型：将 VLA 扩展为 World Model，使模型能预测动作后果并进行模拟推演。
  • 在线学习与自我改进：VLA 模型在部署后根据成功/失败经验自主更新策略。
  • 多机器人泛化：单一模型同时控制不同形态的机器人（单臂、双臂、移动底盘等）。
  • 安全约束集成：在 VLA 推理过程中引入物理安全约束，防止危险动作。

总结

VLA 模型的架构演进遵循一条清晰的主线：从借用 VLM 的离散 Token 范式，到发展出专门面向连续动作空间的生成框架。RT-2 证明了将动作视为语言 Token 的可行性；Octo 和 OpenVLA 推动了开源化和模块化；π0 通过 Flow Matching 打破了离散化瓶颈。每一代模型都在动作表示、推理效率和泛化能力的三角权衡中推进了边界。
