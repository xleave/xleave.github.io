离线强化学习概述：从 BCQ 到 Decision Transformer 的演进

离线强化学习（Offline Reinforcement Learning，也称 Batch RL）从固定的历史数据集中学习策略，不与环境进行在线交互。这一范式的吸引力在于：复用已有数据，避免昂贵且可能危险的在线探索。然而，离线 RL 面临独特的分布偏移挑战，使得标准 RL 算法在离线设定下往往失效。

1. 核心挑战：分布偏移与外推误差

1.1 离线数据的来源

离线数据集 D={(s,a,r,s′)}\mathcal{D} = \{(s, a, r, s')\}D={(s,a,r,s′)} 由行为策略 πβ\pi_\betaπβ​（可能是人类操作者、旧版策略或混合策略）采集。数据集是固定的，学习过程中不会增加新数据。

1.2 分布偏移问题

标准 off-policy RL（如 SAC）在训练中使用 Q 网络估计”在新策略下某动作的未来价值”。如果新策略选择的动作不在训练数据中，Q 网络对该动作的估计是外推（Extrapolation），而非插值。

神经网络的外推极不可靠，Q 值在数据支撑之外的区域通常被严重高估。这导致策略被吸引到 Q 值被高估的区域（即数据集之外的区域），实际执行时性能崩塌。

外推误差∝Qθ(s,aOOD)≫Qθ(s,ain-data)\text{外推误差} \propto Q_\theta(s, a_{\text{OOD}}) \gg Q_\theta(s, a_{\text{in-data}})外推误差∝Qθ​(s,aOOD​)≫Qθ​(s,ain-data​)

2. 约束策略方法

2.1 BCQ（Batch-Constrained Q-learning）

核心思想（Fujimoto et al., 2019）：限制策略仅选择数据集中”可能存在”的动作。

BCQ 使用 CVAE（条件变分自编码器）学习行为策略的动作分布，策略仅从 CVAE 的重建中采样候选动作：

1. CVAE 从数据集中学习 P(a|s) 的近似
2. 策略从 CVAE 采样 N 个候选动作
3. 使用 Q 网络从候选动作中选最优

效果：策略被”约束”在数据集的动作分布附近，避免外推到未知区域。

2.2 CQL（Conservative Q-Learning）

核心思想（Kumar et al., 2020）：对 Q 值施加保守正则化，使数据集之外的动作的 Q 值被系统性压低。

LCQL=α(Ea∼π[Q(s,a)]−Ea∼D[Q(s,a)])+LTD\mathcal{L}_{\text{CQL}} = \alpha \left( \mathbb{E}_{a \sim \pi} [Q(s, a)] - \mathbb{E}_{a \sim \mathcal{D}} [Q(s, a)] \right) + \mathcal{L}_{\text{TD}}LCQL​=α(Ea∼π​[Q(s,a)]−Ea∼D​[Q(s,a)])+LTD​

直觉：正则项惩罚策略选择的动作的高 Q 值，同时保护数据集中动作的 Q 值。结果是数据集之外的动作获得更低的 Q 估计，策略倾向于选择数据集内的动作。

2.3 TD3+BC

核心思想（Fujimoto & Gu, 2021）：在 TD3 的策略更新中简单加入行为克隆正则项。

π=arg⁡max⁡πE(s,a)∼D[λQ(s,π(s))−∥π(s)−a∥2]\pi = \arg\max_\pi \mathbb{E}_{(s,a) \sim \mathcal{D}} \left[ \lambda Q(s, \pi(s)) - \| \pi(s) - a \|^2 \right]π=argmaxπ​E(s,a)∼D​[λQ(s,π(s))−∥π(s)−a∥2]

简洁性：仅一行代码修改，效果却与复杂的离线 RL 方法相当。

2.4 IQL（Implicit Q-Learning）

核心思想（Kostrikov et al., 2021）：完全避免对不在数据集中的动作评估 Q 值，通过隐式学习值函数。

IQL 学习一个条件分位数回归的值函数，仅用数据集中的动作来训练，不需要采样策略之外的动作进行 Q 值评估。

3. 序列建模方法

3.1 范式转变

约束策略方法仍然在 RL 框架内工作（值函数 + 策略梯度）。序列建模方法采取了完全不同的视角：将 RL 重新定义为序列预测问题。

3.2 Decision Transformer（DT）

核心思想（Chen et al., 2021）：将轨迹 (s1,a1,r1,s2,a2,r2,…)(s_1, a_1, r_1, s_2, a_2, r_2, \ldots)(s1​,a1​,r1​,s2​,a2​,r2​,…) 视为一个 Token 序列，使用 Transformer 进行自回归预测。

关键创新是回报条件化（Return-Conditioning）：模型以期望的未来回报 R^t\hat{R}_tR^t​ 作为额外输入，学习在给定期望回报下应采取什么动作。

输入序列: [R̂_1, s_1, a_1, R̂_2, s_2, a_2, ..., R̂_t, s_t]
输出: a_t (预测的动作)

推理时，设定一个高目标回报 R^1\hat{R}_1R^1​，模型生成实现该回报所需的动作序列。

优势：

  • 不需要 Q 值估计，避免了外推误差问题。
  • 直接利用 Transformer 的序列建模能力。
  • 可以利用预训练 Transformer 的架构。

局限：

  • 性能受限于数据集中最优轨迹的质量。
  • 不能通过动态规划”拼接”不同轨迹的优质片段。
  • 在随机环境中，回报条件化可能不稳定。

3.3 Trajectory Transformer

将轨迹中的所有元素（状态、动作、奖励）统一离散化为 Token，使用 Transformer 进行 Beam Search 式规划：

1. 离散化所有维度为 Token
2. Transformer 自回归生成未来轨迹
3. Beam Search 搜索高回报的轨迹分支
4. 执行第一步动作

5. 数据集质量的影响

离线 RL 的性能强烈依赖于数据集的质量和覆盖度：

数据集类型特征算法表现
专家数据集高质量，低多样性行为克隆即可表现良好
混合数据集包含各种质量的数据约束策略方法最有优势
随机数据集完全随机采集，覆盖广所有方法表现较差
不完全覆盖缺少某些状态-动作对外推误差问题最严重

6. 在机器人学中的应用

离线 RL 在机器人学中具有独特价值：

  • 利用历史数据：复用过往实验和人工演示的数据。
  • 避免危险探索：不需要在真机上进行可能导致碰撞的探索。
  • 多源数据融合：合并来自仿真、人工演示和旧策略的异构数据。

实践中的典型流程：

1. 收集多源数据（人工演示 + 旧策略 + 仿真数据）
2. 使用离线 RL 训练初始策略
3. 在真机上少量在线微调（可选）

总结

离线 RL 通过从固定数据集中学习，避免了昂贵的在线交互。其核心挑战——分布偏移导致的外推误差——通过约束策略方法（BCQ、CQL）和序列建模方法（Decision Transformer）以不同方式解决。在机器人学等样本成本高昂的领域，离线 RL 提供了一条利用历史数据提升策略性能的有效途径。
