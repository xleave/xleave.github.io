背景

在个人博客中记录“足迹”（即文章撰写时的物理IP属地或地理坐标）能够为静态内容注入空间维度的联结感。为了保证数据隐私与离线可访问性，博客不采用任何第三方动态地图API（如高德、百度或 Mapbox），而是基于纯 SVG 绘制一个离线的物理足迹轨迹地图。

该地图包含经纬度虚线网格、多城市雷达脉冲闪烁点（Pulsing radar points）以及连接前后足迹的二次贝塞尔曲线轨迹。然而，纯网格点缺乏陆地轮廓作为参照物，容易给访问者带来“地图加载失败”的空白感。为此，需要将一个 CC0 许可的轻量级世界地图矢量 SVG 资源作为底图，嵌套在 SVG 视口内，并使其与我们定义的自定义经纬度映射规则精准对齐。

坐标投影映射算法

在开始对齐世界地图底图前，首先需要确定足迹点的坐标映射算法。我们选定的地图边界（涵盖欧洲/英国到中国）如下：

const mapBounds = {
  minLng: -20,  // 西经 20°
  maxLng: 140, // 东经 140°
  minLat: 10,  // 北纬 10°
  maxLat: 65   // 北纬 65°
};

我们的 SVG 视口（Viewport）大小定义为宽度 W = 800，高度 H = 450。

由于该区域的经纬度跨度较小，在不需要极高精度测绘的场景下，我们可以直接使用简单的**等距矩形投影（Equirectangular projection）**将经纬度线性映射到 SVG 坐标系：

  1. 经度映射 (xxx 轴)： xxx 轴的计算非常直观，经度在 [minLng,maxLng][minLng, maxLng][minLng,maxLng] 范围内线性分布： x=lng−minLngmaxLng−minLng×Wx = \frac{lng - minLng}{maxLng - minLng} \times Wx=maxLng−minLnglng−minLng​×W

  2. 纬度映射 (yyy 轴)： 由于 SVG 的坐标原点 (0,0)(0, 0)(0,0) 在左上角，而纬度越高 yyy 坐标应越小，因此需要用总高度 HHH 减去线性占比： y=H−(lat−minLatmaxLat−minLat×H)y = H - \left( \frac{lat - minLat}{maxLat - minLat} \times H \right)y=H−(maxLat−minLatlat−minLat​×H)

在 Astro 组件中，用 TypeScript 实现的坐标换算函数如下：

const getSvgCoords = (lat: number, lng: number) => {
  const x = ((lng - mapBounds.minLng) / (mapBounds.maxLng - mapBounds.minLng)) * 800;
  const y = 450 - ((lat - mapBounds.minLat) / (mapBounds.maxLat - mapBounds.minLat)) * 450;
  return { x, y };
};

外部底图的精准对齐数学推导

引入世界地图矢量底图（map.svg）时，由于外部 SVG 描绘的是全球范围（经度 [−180,180][-180, 180][−180,180]，纬度 [−90,90][-90, 90][−90,90]），而我们的视口仅展示经度 [−20,140][-20, 140][−20,140]，纬度 [10,65][10, 65][10,65] 的局部区域，因此必须使用 <image> 标签对底图进行精准的缩放（width, height）与偏移（x, y）配置。

3. SVG 嵌套整合

结合上述推导出的几何参数，在 SVG 的最底层（网格虚线下方）插入 <image> 标签：

<svg viewBox="0 0 800 450" class="coordinate-map">
  <!-- 底图背景，使用计算出的对齐参数 -->
  <image 
    href="/img/map.svg" 
    x="-800" 
    y="-204.54" 
    width="1800" 
    height="1472.73" 
    class="map-bg-image" 
  />
  <!-- 网格线、轨迹线与节点在此上方绘制 -->
</svg>

视觉打磨与样式适配

地图底图不应喧宾夺主，需要表现得极为克制。同时，底图需要与博客的配色系统和深色调控制面板相契合。

由于外部 SVG 被引入为 <image> 时无法直接通过 CSS 修改其内部 path 的 fill 或 stroke，我们选择使用高兼容性的 CSS 滤镜（Filter） 组合来控制其呈现颜色：

.map-bg-image {
  pointer-events: none; /* 确保不阻碍下方的雷达点鼠标 hover */
  opacity: 0.12;       /* 保持暗淡，仅提供弱参照 */
  /* 默认底图颜色为黑色，使用 invert 滤镜反转为白灰色，并调整亮度以配合深色面板 */
  filter: invert(1) brightness(0.65) contrast(1.1);
  transition: opacity 0.3s ease, filter 0.3s ease;
}

/* 鼠标悬停在地图区域时，稍微调亮地图轮廓，增强交互反馈 */
.coordinate-map:hover .map-bg-image {
  opacity: 0.18;
  filter: invert(1) brightness(0.8) contrast(1.1);
}

通过这一方案，底图在深色面板上呈现为低饱和度、半透明的灰白色边界，完美契合了博客整体的微光暗色调设计。

总结

通过对等距矩形投影做简单的比例缩放（2.25倍与3.27倍）及像素原点平移（-800px与-204.54px），我们成功在前端离线环境下将世界矢量底图与自定义的局部坐标系完全对齐。这种方案避免了引入数兆大小的第三方地图库，极大地减轻了页面加载负担，同时保持了视觉设计的一致性。
