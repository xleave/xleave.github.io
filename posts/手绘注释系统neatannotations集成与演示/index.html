什么是 Neat Annotations？

Neat Annotations 是一个极简的手绘风格 CSS 注释库，只需为任意行内元素添加几个 CSS 类名，即可自动生成带有手写箭头和手绘标签的视觉批注——完全不依赖 JavaScript，单个 CSS 文件即可工作。

它非常适合在技术博客中对关键术语、公式变量或代码片段进行直观的视觉标注，使读者一眼就能抓住重点。

使用方式：

<span class="ann ann-n ann-blue" data-note="你的批注文字">被标注的文字</span>

其中 ann-n 控制箭头方向（北/上），ann-blue 控制颜色，data-note 是标签内容。

色彩系统

内置 5 种颜色主题 + 一个动态彩虹效果：

默认灰 琥珀色 蓝色 绿色 红色 紫色 彩虹

也可以通过 CSS 自定义属性 --ann-color 设置任意颜色：

<span class="ann ann-n" data-note="自定义颜色" style="--ann-color: #ff1493">热粉红</span>

纯高亮模式

省略 data-note 属性时，注释退化为纯荧光笔高亮效果——非常适合在段落中标记关键词：

在 Sim-to-Real 迁移中，域随机化和系统辨识是两种互补的策略：前者通过随机扰动训练分布来提升策略鲁棒性，后者通过精确估计物理参数来缩小仿真偏差。

示例 3：代码关键行批注

class Controller:
    def compute(self, q_desired, q_actual):
        error = q_desired - q_actual          # 关节位置误差
        torque = self.kp * error              # 比例控制项
        torque += self.gravity_compensation() # 重力前馈补偿
        return np.clip(torque, -τ_max, τ_max) # 力矩饱和限幅

在上面这段代码中，力矩饱和限幅是保护硬件安全的关键步骤，而重力前馈补偿能显著提升控制精度。

安装与使用

本站已将 neat-annotations.css 下载至本地并全局引入，所有 Markdown/MDX 文章中均可直接使用：

<!-- 基本用法 -->
<span class="ann ann-方向 ann-颜色" data-note="标注内容">被注释文字</span>

<!-- 纯高亮 -->
<span class="ann ann-amber">高亮文字</span>

<!-- 自定义颜色 -->
<span class="ann ann-n" data-note="批注" style="--ann-color: #00c2d7">目标</span>
参数说明可选值
方向类箭头+标签的放置方位ann-n, ann-ne, ann-e, ann-se, ann-s, ann-sw, ann-w, ann-nw
颜色类预设色调ann-amber, ann-blue, ann-green, ann-red, ann-purple, ann-rainbow
data-note标签文字内容任意字符串（省略则只显示高亮）
ann-no-mark移除高亮背景用于嵌套时避免双重高亮
--ann-colorCSS 自定义颜色任意 CSS 颜色值
--ann-label-max-width标签最大宽度如 200px、15em
