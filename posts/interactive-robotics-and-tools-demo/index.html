1. 2-DOF 机械臂逆运动学求解器 (IK Simulator)

在机械臂控制中，逆运动学 (Inverse Kinematics, IK) 是指已知机械臂末端执行器 (End-Effector) 的目标位姿 (x,y)(x, y)(x,y)，求解关节电机旋转角度 θ1\theta_1θ1​ 与 θ2\theta_2θ2​ 的数学过程。

对于简易的二自由度平面机械臂（臂长分别为 L1L_1L1​ 与 L2L_2L2​），我们可以采用代数法进行解析求解：

cos⁡θ2=x2+y2−L12−L222L1L2\cos\theta_2 = \frac{x^2 + y^2 - L_1^2 - L_2^2}{2 L_1 L_2}cosθ2​=2L1​L2​x2+y2−L12​−L22​​

依据三角恒等式解出 θ2\theta_2θ2​（正负号分别对应肘部向上/Elbow Up 和肘部向下/Elbow Down 两种构型）：

θ2=±arccos⁡(cos⁡θ2)\theta_2 = \pm \arccos(\cos\theta_2)θ2​=±arccos(cosθ2​)

利用 θ2\theta_2θ2​ 进一步解出肩膀关节角 θ1\theta_1θ1​：

θ1=atan2(y,x)−atan2(L2sin⁡θ2,L1+L2cos⁡θ2)\theta_1 = \text{atan2}(y, x) - \text{atan2}(L_2 \sin\theta_2, L_1 + L_2 \cos\theta_2)θ1​=atan2(y,x)−atan2(L2​sinθ2​,L1​+L2​cosθ2​)

2. 路径规划与网格寻路可视化 (Pathfinding Visualizer)

路径规划（Path Planning）是移动机器人自主导航与 SLAM（同时定位与地图构建）的关键部分。常用的经典格栅图规划算法包括 Dijkstra 和 A* 算法。

  • Dijkstra 算法：经典的无启发式单源最短路径算法，从起点向外均匀扩展搜索波面，能保证找到最优路径，但搜索效率较低。
  • A* 算法：基于启发式估价函数 f(n)=g(n)+h(n)f(n) = g(n) + h(n)f(n)=g(n)+h(n) 的启发式搜索算法。其中 g(n)g(n)g(n) 是从起点到当前网格的实际代价，h(n)h(n)h(n) 是估算当前网格到目标网格的曼哈顿距离或欧氏距离。

您可以在下方网格中直接点击并拖拽鼠标绘制障碍墙，或拖动 S（起点）与 G（终点）图标，点击“开始规划”观看算法实时波面展开与寻路动画。

3. 3D 旋转矩阵与四元数可视化 (3D Rotations Visualizer)

在三维空间中描述刚体姿态（Attitude Representation），通常使用旋转矩阵 SO(3)SO(3)SO(3)、欧拉角（Yaw-偏航, Pitch-俯仰, Roll-翻滚）或四元数（Quaternion）。

欧拉角容易理解，但容易在 Pitch 达到 ±90∘\pm 90^\circ±90∘ 时陷入奇异性崩溃（Gimbal Lock, 步进死锁）。因此在底层动力学解算和卡尔曼滤波中，更倾向于采用无奇异的单位四元数 q=[qw,qx,qy,qz]q = [q_w, q_x, q_y, q_z]q=[qw​,qx​,qy​,qz​]。

本组件基于正交等轴测投影，将 ZZZ 轴向上（蓝色）、XXX 轴前右（红色）、YYY 轴后左（绿色）的 3D 三轴坐标系，投影至 2D 平面：

Xscreen=(X3d−Y3d)⋅cos⁡(30∘)X_{\text{screen}} = (X_{3d} - Y_{3d}) \cdot \cos(30^\circ)Xscreen​=(X3d​−Y3d​)⋅cos(30∘) Yscreen=(X3d+Y3d)⋅sin⁡(30∘)−Z3dY_{\text{screen}} = (X_{3d} + Y_{3d}) \cdot \sin(30^\circ) - Z_{3d}Yscreen​=(X3d​+Y3d​)⋅sin(30∘)−Z3d​

4. 浏览器 Web Serial API 调试控制台 (Web Serial Console)

Web Serial API 是现代 Web 的一项新标准，允许网站通过浏览器直接与本地串口设备（如 Arduino、ESP32、微控制器 STM32）进行双向串口通讯，无需安装任何专用客户端或中间件。

对于具身智能设备，在调试执行机构的底层伺服控制电路（PID 调谐）时，能够直接通过网页控制台收发指令将极大加快迭代效率。

注意：请使用基于 Chromium 内核的最新版 Chrome 或 Edge 浏览器以启用本功能。

6. 学术论文极速闪阅模式 (RSVP Speed Reader)

对于科技工作者，快速扫读大量的 arXiv 会议论文（如关于 VLA 或 Diffusion Policy 的前沿进展）是一项高频需求。

RSVP (Rapid Serial Visual Presentation) 即快速串行视觉呈现。它通过在屏幕中央以极高速度（例如 300 WPM - 800 WPM）轮流闪烁展示文本单词，并且将单词的最佳识别点 (Optimal Recognition Point, ORP) 在视线焦点处严格垂直对齐（即单词中间特定的某个字母以高亮红色居中对齐）。这使得眼球无需左右扫视移动，直接通过黄斑中心凹快速捕获词义，能使阅读速度提升 2 至 3 倍。
