多任务强化学习中的参数共享与任务干扰问题

多任务强化学习（Multi-Task RL）的目标是训练单一策略同时掌握多种任务。参数共享（Parameter Sharing）是实现这一目标的常见策略——多个任务共享同一组网络参数，通过任务条件化（如 Task ID 或语言指令）区分不同任务。

然而，参数共享在带来正向迁移（Positive Transfer）的同时，也可能引发任务间的负向干扰（Negative Interference）。本文分析这一矛盾及其缓解方法。

1. 参数共享的动机

1.1 正向迁移

不同任务之间存在共享的底层特征：

  • 视觉特征：不同操作任务共享物体识别和空间推理能力。
  • 运动原语：接近物体、抓取、放置等基础运动技能可复用。
  • 物理直觉：重力、摩擦等物理约束在所有任务中通用。

参数共享使这些通用知识在任务间自动迁移，减少了总训练数据需求。

1.2 部署效率

单一多任务模型比多个独立任务模型更容易部署和维护——更小的显存占用、统一的推理管线以及更简单的版本管理。

2. 任务干扰问题

2.1 梯度冲突

当两个任务的损失函数产生方向相反的梯度时，参数更新会在两者之间折中，可能导致两个任务的性能都下降：

gA=∇θLA,gB=∇θLBg_A = \nabla_\theta \mathcal{L}_A, \quad g_B = \nabla_\theta \mathcal{L}_BgA​=∇θ​LA​,gB​=∇θ​LB​

若 gA⋅gB<0g_A \cdot g_B < 0gA​⋅gB​<0（余弦相似度为负），则两个任务的梯度方向冲突。

2.2 表征干扰

不同任务可能需要网络学习不同甚至矛盾的特征表示。例如：

  • 任务 A（精密插入）需要关注细粒度的空间对齐。
  • 任务 B（物体分类抓取）需要关注物体的类别语义。

共享的特征提取层被迫在两种需求之间妥协。

2.3 奖励尺度差异

不同任务的奖励尺度可能差异很大。奖励尺度大的任务主导梯度更新，小尺度任务被”淹没”。

3. 梯度冲突的检测与缓解

3.1 PCGrad（Projecting Conflicting Gradients）

原理（Yu et al., 2020）：当两个任务的梯度冲突时，将一个任务的梯度投影到另一个任务梯度的法平面上，消除冲突分量：

投影后的梯度 g~A\tilde{g}_Ag~​A​ 与 gBg_BgB​ 正交，不再互相干扰。

3.2 GradNorm

原理（Chen et al., 2018）：自动调整各任务损失的权重，使所有任务的梯度范数保持平衡：

梯度范数较小的任务获得更大的权重，防止被其他任务淹没。

3.3 CAGrad（Conflict-Averse Gradient）

原理：在多个任务梯度的凸包中寻找一个方向，使其在所有任务上的最小下降量最大化：

这确保更新方向对所有任务都”至少有一些进步”。

4. 网络架构层面的缓解

4.1 Soft Modularization

使用路由网络（Router）根据任务动态选择模块组合：

输入 → [模块 1] [模块 2] [模块 3] ... → 路由器选择 → 输出

不同任务使用不同的模块组合，共享有用的模块但隔离冲突的模块。

4.2 Multi-Head 架构

共享 Backbone，每个任务有独立的输出头（Head）：

共享 Backbone → 共享特征
                  ↓
        ┌─── Task A Head ──→ 动作 A
        ├─── Task B Head ──→ 动作 B
        └─── Task C Head ──→ 动作 C

特征提取层在任务间共享，策略输出层独立。这是正向迁移和干扰隔离的折中方案。

4.3 Mixture of Experts (MoE)

使用门控网络根据输入动态分配计算到不同专家：

y=∑igi(x)⋅Ei(x)y = \sum_i g_i(x) \cdot E_i(x)y=∑i​gi​(x)⋅Ei​(x)

不同任务可能激活不同的专家，实现隐式的任务分离。

5. 任务分组

5.1 动机

并非所有任务都适合共享参数。识别哪些任务之间存在正向迁移、哪些存在负向干扰，可以指导分组策略。

5.2 基于梯度相似度的分组

计算任务间梯度的余弦相似度矩阵，将梯度相似度高的任务分为一组共享参数：

similarity_matrix[i][j] = cosine_sim(grad_i, grad_j)
groups = cluster(similarity_matrix, threshold=0)

5.3 基于性能影响的分组

通过消融实验评估每对任务的联合训练是否优于独立训练。如果联合训练性能下降，则分开训练。

6. 奖励归一化

7. 在机器人学中的实践

7.1 典型场景

  • 多物体抓取：同一策略抓取不同形状的物体。
  • 多技能策略：抓取、放置、推动、旋转共享一个策略。
  • VLA 多任务训练：单一 VLA 模型通过语言条件化执行多种任务。

7.2 实践建议

建议说明
从 Multi-Head 开始共享 Backbone + 独立 Head 是简单有效的起点
监控任务间梯度余弦相似度负值意味着干扰，需要采取措施
使用 Pop-Art 归一化自动处理奖励尺度差异
语言条件化替代 Task ID语言嵌入提供更丰富的任务表示，促进迁移
关注尾部任务性能整体性能提升可能掩盖个别任务的退化

总结

多任务参数共享是一把双刃剑：正向迁移加速学习，负向干扰降低性能。梯度冲突是干扰的直接表现，可通过 PCGrad、GradNorm 等方法在优化层面缓解。网络架构层面的模块化设计（Multi-Head、MoE、Soft Modularization）提供了更根本的隔离方案。在实践中，监控任务间的梯度相似度、使用奖励归一化、并根据干扰程度进行任务分组，是构建高效多任务机器人策略的关键。
