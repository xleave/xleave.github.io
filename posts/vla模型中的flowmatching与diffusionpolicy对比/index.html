VLA 模型中的 Flow Matching 与 Diffusion Policy 对比

在 VLA 模型的动作生成环节，Diffusion Policy 和 Flow Matching 是两种主流的连续动作生成范式。它们都属于基于去噪的生成模型，通过将噪声逐步转化为有意义的动作序列来实现策略输出。然而两者在数学基础、训练目标和推理效率上存在显著差异。

本文从原理层面对比这两种方法，分析其在机器人策略学习中的工程权衡。

1. 背景：为什么使用生成模型做动作输出？

传统的策略学习方法（如行为克隆）通常使用确定性回归或高斯分布输出动作。但在真实的操作任务中，存在多模态动作分布的问题：

  • 绕过障碍物时，可以从左侧或右侧绕行。
  • 抓取物体时，可以从不同角度接近。
  • 放置物体时，可以选择不同的放置位置。

确定性回归会输出各模式的平均值（一个无效的中间动作），高斯分布无法表达多峰结构。Diffusion/Flow Matching 等生成模型能天然建模多模态分布，因此成为 VLA 动作输出的首选方案。

2. Diffusion Policy

2.1 数学框架

Diffusion Policy 基于去噪扩散概率模型（DDPM, Denoising Diffusion Probabilistic Models）。核心思想是定义一个前向扩散过程，将干净数据 x0\mathbf{x}_0x0​（动作序列）逐步加噪至纯噪声 xT∼N(0,I)\mathbf{x}_T \sim \mathcal{N}(0, \mathbf{I})xT​∼N(0,I)，然后训练一个网络学习反向去噪过程。

前向过程（固定，不含可学习参数）：

q(xt∣x0)=N(xt;αˉtx0,(1−αˉt)I)q(\mathbf{x}_t | \mathbf{x}_0) = \mathcal{N}(\mathbf{x}_t; \sqrt{\bar{\alpha}_t} \mathbf{x}_0, (1 - \bar{\alpha}_t) \mathbf{I})q(xt​∣x0​)=N(xt​;αˉt​​x0​,(1−αˉt​)I)

其中 αˉt=∏s=1tαs\bar{\alpha}_t = \prod_{s=1}^{t} \alpha_sαˉt​=∏s=1t​αs​ 是噪声调度（Noise Schedule）的累积系数。

反向过程（可学习）：

pθ(xt−1∣xt)=N(xt−1;μθ(xt,t),σt2I)p_\theta(\mathbf{x}_{t-1} | \mathbf{x}_t) = \mathcal{N}(\mathbf{x}_{t-1}; \mu_\theta(\mathbf{x}_t, t), \sigma_t^2 \mathbf{I})pθ​(xt−1​∣xt​)=N(xt−1​;μθ​(xt​,t),σt2​I)

2.2 训练目标

Diffusion Policy 训练一个噪声预测网络 ϵθ\epsilon_\thetaϵθ​，目标是预测添加到数据中的噪声：

Ldiffusion=Et,x0,ϵ[∥ϵ−ϵθ(xt,t,c)∥2]\mathcal{L}_{\text{diffusion}} = \mathbb{E}_{t, \mathbf{x}_0, \epsilon} \left[ \| \epsilon - \epsilon_\theta(\mathbf{x}_t, t, c) \|^2 \right]Ldiffusion​=Et,x0​,ϵ​[∥ϵ−ϵθ​(xt​,t,c)∥2]

其中 ccc 是条件信息（视觉观测、语言指令等），ϵ∼N(0,I)\epsilon \sim \mathcal{N}(0, \mathbf{I})ϵ∼N(0,I)。

2.3 推理过程

从纯噪声 xT∼N(0,I)\mathbf{x}_T \sim \mathcal{N}(0, \mathbf{I})xT​∼N(0,I) 出发，执行 TTT 步去噪迭代：

xt−1=1αt(xt−1−αt1−αˉtϵθ(xt,t,c))+σtz\mathbf{x}_{t-1} = \frac{1}{\sqrt{\alpha_t}} \left( \mathbf{x}_t - \frac{1 - \alpha_t}{\sqrt{1 - \bar{\alpha}_t}} \epsilon_\theta(\mathbf{x}_t, t, c) \right) + \sigma_t \mathbf{z}xt−1​=αt​​1​(xt​−1−αˉt​​1−αt​​ϵθ​(xt​,t,c))+σt​z

典型的去噪步数 T=100∼1000T = 100 \sim 1000T=100∼1000（DDPM），可通过 DDIM 等加速方法减少至 10∼5010 \sim 5010∼50 步。

3. Flow Matching

3.1 数学框架

Flow Matching（FM）基于连续归一化流（Continuous Normalizing Flows, CNF）。核心思想是学习一个速度场 vθ(x,t)v_\theta(\mathbf{x}, t)vθ​(x,t)，定义从噪声分布 p0=N(0,I)p_0 = \mathcal{N}(0, \mathbf{I})p0​=N(0,I) 到数据分布 p1=pdatap_1 = p_{\text{data}}p1​=pdata​ 的连续变换。

数据点沿速度场的轨迹由常微分方程（ODE）描述：

dxdt=vθ(x(t),t),t∈[0,1]\frac{d\mathbf{x}}{dt} = v_\theta(\mathbf{x}(t), t), \quad t \in [0, 1]dtdx​=vθ​(x(t),t),t∈[0,1]

其中 x(0)∼p0\mathbf{x}(0) \sim p_0x(0)∼p0​（噪声），x(1)∼p1\mathbf{x}(1) \sim p_1x(1)∼p1​（数据）。

3.2 条件流匹配（Conditional Flow Matching）

直接学习全局速度场是困难的。Flow Matching 的关键技巧是使用条件流匹配（CFM）：对每个数据点 x1\mathbf{x}_1x1​，定义一条从噪声 x0\mathbf{x}_0x0​ 到数据 x1\mathbf{x}_1x1​ 的简单条件路径（通常是线性插值）：

xt=(1−t)x0+tx1\mathbf{x}_t = (1 - t) \mathbf{x}_0 + t \mathbf{x}_1xt​=(1−t)x0​+tx1​

对应的条件速度场为：

ut(x∣x1)=x1−x0u_t(\mathbf{x} | \mathbf{x}_1) = \mathbf{x}_1 - \mathbf{x}_0ut​(x∣x1​)=x1​−x0​

3.3 训练目标

训练网络 vθv_\thetavθ​ 匹配条件速度场：

LFM=Et,x0,x1[∥vθ(xt,t,c)−(x1−x0)∥2]\mathcal{L}_{\text{FM}} = \mathbb{E}_{t, \mathbf{x}_0, \mathbf{x}_1} \left[ \| v_\theta(\mathbf{x}_t, t, c) - (\mathbf{x}_1 - \mathbf{x}_0) \|^2 \right]LFM​=Et,x0​,x1​​[∥vθ​(xt​,t,c)−(x1​−x0​)∥2]

其中 t∼U(0,1)t \sim \mathcal{U}(0, 1)t∼U(0,1)，x0∼N(0,I)\mathbf{x}_0 \sim \mathcal{N}(0, \mathbf{I})x0​∼N(0,I)，x1∼pdata\mathbf{x}_1 \sim p_{\text{data}}x1​∼pdata​。

注意，训练目标的形式与 Diffusion 的噪声预测目标非常相似，但预测的对象从”噪声”变为”速度”（即从噪声到数据的方向向量）。

3.4 推理过程

使用 ODE 求解器从噪声积分至数据：

x1=x0+∫01vθ(x(t),t,c) dt\mathbf{x}_1 = \mathbf{x}_0 + \int_0^1 v_\theta(\mathbf{x}(t), t, c) \, dtx1​=x0​+∫01​vθ​(x(t),t,c)dt

实际实现中使用离散化的欧拉法（Euler Method）：

xt+Δt=xt+Δt⋅vθ(xt,t,c)\mathbf{x}_{t+\Delta t} = \mathbf{x}_t + \Delta t \cdot v_\theta(\mathbf{x}_t, t, c)xt+Δt​=xt​+Δt⋅vθ​(xt​,t,c)

步数通常为 5∼205 \sim 205∼20 步。

4. 核心对比

4.1 数学本质

维度Diffusion PolicyFlow Matching
理论基础随机微分方程（SDE）常微分方程（ODE）
前向过程逐步加噪（马尔可夫链）线性插值（直线路径）
学习目标预测噪声 ϵ\epsilonϵ预测速度场 vvv
生成路径曲线（噪声调度决定）直线（线性插值）
随机性SDE 采样包含随机项ODE 采样是确定性的

4.2 推理效率

Flow Matching 的核心优势在于推理效率。由于其传输路径是线性的（直线），ODE 的数值积分更容易，所需的离散化步数更少：

指标Diffusion Policy (DDIM)Flow Matching
典型去噪步数10-505-20
每步计算量1 次前向传播1 次前向传播
总推理时间中等较低
动作质量步数过少时退化5 步即可获得较好质量

4.3 训练稳定性

维度Diffusion PolicyFlow Matching
噪声调度敏感性高（需要仔细选择 β\betaβ 调度）低（无噪声调度）
损失函数方差较高（不同 ttt 的损失量级不同）较低（线性路径的目标更稳定）
训练收敛速度中等较快

4.4 多模态分布建模

两种方法都能建模多模态动作分布，但机制不同：

  • Diffusion Policy：通过 SDE 的随机性，不同的噪声采样会引导至分布的不同模式。
  • Flow Matching：通过 ODE 的确定性路径，不同的初始噪声采样确定性地映射至不同模式。

实践中，Flow Matching 的确定性生成在需要可复现结果的场景中更有优势。

5. 在 VLA 模型中的应用实例

5.1 Diffusion Policy（Chi et al., 2023）

Diffusion Policy 是最早将扩散模型用于机器人策略学习的代表工作。它使用 U-Net 或 Transformer 作为去噪网络，以视觉观测为条件生成 Action Chunk：

条件: 视觉编码 c = Encoder(image)
输入: 噪声动作 x_T ~ N(0, I), shape: (H, D)
      ↓ DDIM 去噪 × 10-50 步
输出: 干净动作 x_0, shape: (H, D)

5.2 π0（Black et al., 2024）

π0 选择 Flow Matching 作为动作生成方法，通过 Action Expert 中的 Flow Matching Head 生成动作：

条件: VLM 特征 + 动作专家交叉注意力
输入: 噪声动作 Token x_0 ~ N(0, I)
      ↓ 欧拉法积分 × 10 步
输出: 干净动作 Token x_1 → 线性投影 → 连续动作

π0 选择 Flow Matching 的核心原因：

  1. 推理延迟：Flow Matching 需要的去噪步数更少（10 步 vs. Diffusion 的 50 步），在延迟敏感的机器人控制场景中更有优势。
  2. 训练简洁性：无需噪声调度设计，训练目标更直接。
  3. 与 VLM 集成：线性插值路径的简洁性使其更容易与预训练 VLM Backbone 联合训练。

6. 工程实现差异

6.1 网络架构

两种方法可以使用相同的骨干网络（如 U-Net、Transformer），差异仅在输入/输出的定义：

# Diffusion: 网络预测噪声
predicted_noise = model(noisy_action, timestep, condition)
loss = MSE(predicted_noise, actual_noise)

# Flow Matching: 网络预测速度
predicted_velocity = model(interpolated_action, time, condition)
target_velocity = clean_action - noise
loss = MSE(predicted_velocity, target_velocity)

6.2 采样代码

# Diffusion (DDIM)
x = torch.randn(batch, H, D)  # 纯噪声
for t in reversed(timesteps):
    predicted_noise = model(x, t, condition)
    x = ddim_step(x, predicted_noise, t)

# Flow Matching (Euler)
x = torch.randn(batch, H, D)  # 纯噪声
dt = 1.0 / num_steps
for t in torch.linspace(0, 1, num_steps):
    velocity = model(x, t, condition)
    x = x + dt * velocity

7. 选型建议

场景推荐方法原因
推理延迟敏感（实时控制）Flow Matching更少的去噪步数
已有 Diffusion 训练基础设施Diffusion Policy迁移成本低
需要高质量多模态分布两者均可差异不大
需要可复现的确定性输出Flow MatchingODE 采样是确定性的
大规模预训练 + 微调Flow Matching训练更稳定

总结

Diffusion Policy 和 Flow Matching 是解决同一问题（从条件分布中生成连续动作序列）的两条技术路线。Diffusion 基于 SDE 框架，理论更成熟、生态更完善；Flow Matching 基于 ODE 框架，训练更简洁、推理更高效。在 VLA 模型的演进中，Flow Matching 正因其推理效率优势而成为更受青睐的选择，π0 的成功验证了这一趋势。
