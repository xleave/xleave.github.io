Sim-to-Real Gap 的来源分类与缓解策略总览

在仿真中训练机器人策略并迁移到真实世界（Sim-to-Real Transfer）是降低训练成本和加速研究迭代的关键路径。然而，仿真环境与真实环境之间不可避免地存在差异——即 Sim-to-Real Gap。这种差异可能导致在仿真中表现优异的策略在真机上完全失效。

本文系统分类 Sim-to-Real Gap 的来源，并综述各类缓解策略。

1. Gap 的来源分类

1.1 动力学 Gap（Dynamics Gap）

仿真器的物理引擎对真实世界动力学的建模是近似的。常见的误差源包括：

误差源说明
接触模型仿真中的接触力通常使用弹簧-阻尼模型近似，缺乏真实接触的复杂性（如微观形变、摩擦各向异性）
摩擦系数仿真中通常使用固定的库仑摩擦系数，真实摩擦受材料、湿度、温度影响
关节动力学真实关节存在间隙（backlash）、弹性形变和非线性摩擦，仿真中常简化或忽略
柔性体布料、绳索等柔性物体的仿真精度远低于刚体
流体动力学倒水等涉及流体的操作在仿真中极难准确建模

1.2 视觉 Gap（Visual Gap）

仿真渲染的图像与真实相机拍摄的图像存在视觉差异：

误差源说明
光照模型仿真通常使用简化的光照模型（如 Phong），缺乏真实的全局光照、焦散和软阴影
材质纹理仿真物体的材质和纹理与真实物体不完全匹配
相机噪声真实相机存在传感器噪声、运动模糊、自动曝光变化
背景差异仿真背景与真实环境的背景不同

1.3 传感器 Gap（Sensor Gap）

除视觉外，其他传感器也存在仿真与真实的差异：

  • 深度传感器：真实深度图有噪声、空洞和多路径干扰，仿真深度图通常是理想的。
  • 力/力矩传感器：真实传感器有漂移和噪声，仿真中通常给出精确值。
  • 关节编码器：真实编码器有量化误差和延迟。

1.4 执行器 Gap（Actuator Gap）

仿真中的执行器模型通常是理想化的：

  • 延迟：真实系统的通信延迟（命令从发出到执行）在仿真中常被忽略。
  • 饱和：真实电机有力矩和速度上限，仿真中可能未正确建模。
  • 滞后：真实执行器存在响应滞后，仿真中的响应通常是瞬时的。

2. Gap 的量化

在设计缓解策略之前，量化 Gap 的大小和类型有助于确定优先级：

2.1 动力学参数对比

在仿真和真实环境中执行相同的开环动作序列，对比：

  • 关节轨迹的偏差：Δq(t)=qreal(t)−qsim(t)\Delta q(t) = q_{\text{real}}(t) - q_{\text{sim}}(t)Δq(t)=qreal​(t)−qsim​(t)
  • 末端执行器位姿偏差
  • 接触力的时序对比

2.2 视觉差异度量

使用 Fréchet Inception Distance（FID）或其他分布距离度量仿真渲染图像与真实图像的分布差异。

2.3 策略性能差

最直接的量化：相同策略在仿真中的成功率 SRsimSR_{\text{sim}}SRsim​ 与真机中的成功率 SRrealSR_{\text{real}}SRreal​ 之差。

Gap=SRsim−SRreal\text{Gap} = SR_{\text{sim}} - SR_{\text{real}}Gap=SRsim​−SRreal​

3. 缓解策略

3.1 域随机化（Domain Randomization）

核心思想：在仿真训练中随机化环境参数（物理参数、视觉外观等），使策略对参数变化具有鲁棒性。如果随机化范围覆盖了真实参数，则策略能泛化到真实环境。

详细分析见专题文章。

3.2 系统辨识（System Identification）

核心思想：精确测量真实环境的物理参数，在仿真中复现这些参数，减小仿真与真实之间的参数差异。

详细分析见专题文章。

3.3 域适应（Domain Adaptation）

核心思想：通过学习仿真域和真实域之间的映射，在特征空间或像素空间中对齐两个域的分布。

方法包括：

  • 视觉域适应：使用 GAN 或风格迁移将仿真图像转化为真实风格，或将真实图像转化为仿真风格。
  • 特征对齐：通过对抗训练使编码器提取的特征在仿真和真实域中不可区分。
  • 不变表示学习：学习对域差异不变的特征表示。

3.4 渐进式迁移

核心思想：不直接从仿真跳到真实，而是通过中间步骤逐步缩小 Gap：

  1. 在简单仿真中预训练。
  2. 在高保真仿真中微调。
  3. 在真实世界中少量微调。

3.5 真实数据微调

最直接的方法：在仿真中预训练后，使用少量真实数据对策略进行微调。这需要在真实环境中收集数据，但数据量远少于从头训练。

4. 不同 Gap 类型的缓解策略匹配

Gap 类型推荐缓解策略
动力学参数不精确域随机化 + 系统辨识
视觉外观差异视觉域随机化 + 域适应
传感器噪声差异传感器噪声注入（仿真中添加模拟噪声）
执行器延迟在仿真中注入随机延迟
柔性物体建模提升仿真器保真度 / 真实数据微调

5. 工程实践建议

5.1 先量化再优化

在投入大量精力优化 Sim2Real 迁移之前，先量化各类 Gap 的相对大小。可能动力学 Gap 是主要矛盾，而视觉 Gap 影响很小（或反之）。

5.2 组合使用多种策略

实际项目中，通常需要组合使用多种缓解策略。例如：

  • 动力学域随机化 + 视觉域随机化 + 传感器噪声注入 + 少量真实数据微调。

5.3 关注延迟建模

通信延迟是最容易被忽略但影响显著的 Gap 来源之一。在仿真中注入 1-3 步的随机观测-动作延迟，可以显著提升迁移鲁棒性。

总结

Sim-to-Real Gap 是多维度的，涵盖动力学、视觉、传感器和执行器等方面。没有单一的缓解策略能解决所有问题。理解 Gap 的具体来源、量化其影响，然后针对性地组合使用域随机化、系统辨识、域适应和真实数据微调等方法，是成功 Sim2Real 迁移的工程方法论。
