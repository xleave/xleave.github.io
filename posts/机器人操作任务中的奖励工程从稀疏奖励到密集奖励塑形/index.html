机器人操作任务中的奖励工程：从稀疏奖励到密集奖励塑形

在强化学习驱动的机器人操作任务中，奖励函数（Reward Function）定义了”什么是好的行为”。然而，设计一个既能引导策略学习、又不引入偏见的奖励函数，是强化学习工程中最具挑战性的环节之一。

本文从稀疏奖励的困难出发，逐步讨论密集奖励设计、势函数奖励塑形以及学习式奖励等方案。

1. 稀疏奖励：简单但难以学习

1.1 定义

稀疏奖励在任务成功时给予固定正奖励，其余时间步奖励为零：

r(s,a)={1if task completed0otherwiser(s, a) = \begin{cases} 1 & \text{if task completed} \\ 0 & \text{otherwise} \end{cases}r(s,a)={10​if task completedotherwise​

1.2 优势

  • 无偏性：不对”如何完成任务”施加先验偏见，策略可以自由发现最优行为。
  • 定义简洁：仅需定义成功条件，无需复杂的奖励工程。

1.3 困难

稀疏奖励下，智能体在探索初期几乎不可能偶然触发成功条件。以”将杯子放到指定位置”为例：

  • 动作空间为 7 维连续空间（6 维末端位姿 + 1 维夹爪）。
  • 每个 episode 包含数百个时间步。
  • 随机探索命中成功条件的概率极低。

没有梯度信号引导，策略更新本质上是随机游走。这被称为探索稀疏性问题。

2. 密集奖励：逐步引导策略

2.1 基于距离的奖励

最直觉的密集奖励设计是：奖励与目标的距离成反比。

r(s)=−∥pee−pgoal∥2r(s) = -\| p_{\text{ee}} - p_{\text{goal}} \|_2r(s)=−∥pee​−pgoal​∥2​

其中 peep_{\text{ee}}pee​ 是末端执行器位置，pgoalp_{\text{goal}}pgoal​ 是目标位置。

优势：提供持续的梯度信号，引导策略向目标移动。

问题：

  • 局部最优：策略可能学到将末端执行器移向目标但不执行抓取的行为。
  • 维度缺失：仅考虑位置，忽略姿态、力等其他重要维度。
  • 尺度问题：不同维度的距离量级可能相差很大。

2.2 多阶段分解奖励

将任务分解为若干阶段，每个阶段设计独立的奖励：

def reward(state):
    r = 0
    # 阶段 1：接近物体
    dist_to_object = distance(ee, object)
    r += -1.0 * dist_to_object

    # 阶段 2：抓取物体
    if is_grasping(state):
        r += 2.0

    # 阶段 3：将物体移向目标
    if is_grasping(state):
        dist_to_goal = distance(object, goal)
        r += -1.0 * dist_to_goal

    # 阶段 4：放置成功
    if is_placed(state):
        r += 10.0

    return r

优势：分阶段引导，策略能逐步学习完整行为。

问题：

  • 权重调优：各阶段奖励的相对权重需要仔细调优。
  • 奖励黑客（Reward Hacking）：策略可能找到最大化某一阶段奖励但不完成任务的捷径。
  • 人工偏见：阶段划分和奖励设计注入了设计者的先验假设。

3. 势函数奖励塑形（Potential-Based Reward Shaping）

3.1 理论基础

Ng et al. (1999) 证明了一个重要定理：如果塑形奖励（Shaping Reward）具有如下势函数（Potential Function）形式，则不会改变最优策略：

F(s,a,s′)=γΦ(s′)−Φ(s)F(s, a, s') = \gamma \Phi(s') - \Phi(s)F(s,a,s′)=γΦ(s′)−Φ(s)

其中 Φ(s)\Phi(s)Φ(s) 是状态 sss 的势函数，γ\gammaγ 是折扣因子。

3.2 直觉

势函数奖励塑形的本质是：给予智能体”朝高势区域移动”的即时奖励，但确保这个额外奖励不改变最优策略。它类似于重力场中势能的变化——从高处移到低处释放势能，但势能差不改变最终的目标位置。

3.3 实践中的设计

将距离的负数作为势函数：

Φ(s)=−∥pee(s)−pgoal∥2\Phi(s) = -\| p_{\text{ee}}(s) - p_{\text{goal}} \|_2Φ(s)=−∥pee​(s)−pgoal​∥2​

则塑形奖励为：

F=γ(−∥pee′−pgoal∥)−(−∥pee−pgoal∥)=∥pee−pgoal∥−γ∥pee′−pgoal∥F = \gamma(-\|p'_{\text{ee}} - p_{\text{goal}}\|) - (-\|p_{\text{ee}} - p_{\text{goal}}\|) = \|p_{\text{ee}} - p_{\text{goal}}\| - \gamma \|p'_{\text{ee}} - p_{\text{goal}}\|F=γ(−∥pee′​−pgoal​∥)−(−∥pee​−pgoal​∥)=∥pee​−pgoal​∥−γ∥pee′​−pgoal​∥

当 γ≈1\gamma \approx 1γ≈1 时，这近似于”距离减少量”——每一步缩短距离得正奖励，增加距离得负奖励。

优势：理论上保证不改变最优策略，同时提供密集的学习信号。

4. Hindsight Experience Replay (HER)

4.1 核心思想

HER（Andrychowicz et al., 2017）从一个不同的角度解决稀疏奖励问题：不改变奖励函数，而是修改经验回放中的目标。

对于一条失败的轨迹（未到达目标 ggg），HER 将轨迹中实际到达的某个状态 s′s's′ 重新标记为”目标”，这样原本的失败轨迹变成了”到达 s′s's′“的成功轨迹。

4.2 算法流程

1. 使用策略执行一条轨迹：(s_0, a_0, s_1, a_1, ..., s_T)，目标为 g
2. 存储原始经验：(s_t, a_t, r_t, s_{t+1}, g) 其中 r_t = 0（失败）
3. 选择替代目标 g' = s_T（轨迹终点）
4. 重新计算奖励：r'_t = reward(s_{t+1}, g') = 1 if s_{t+1} ≈ g'
5. 存储 hindsight 经验：(s_t, a_t, r'_t, s_{t+1}, g')

4.3 适用条件

HER 适用于目标条件化（Goal-Conditioned）策略，且任务的成功判定仅依赖于最终状态与目标的距离。

5. 学习式奖励

5.1 逆强化学习（Inverse RL）

学到的奖励应使专家策略的累积奖励高于其他策略。

优势：无需手工设计奖励。

局限：需要专家演示；学到的奖励可能不唯一且难以解释。

5.2 VLM 作为奖励函数

近年来的研究探索使用视觉-语言模型（VLM）作为奖励源：

def vlm_reward(image, language_goal):
    # 使用 CLIP 计算图像与目标描述的匹配度
    similarity = clip.similarity(image, language_goal)
    return similarity

优势：利用 VLM 的广泛世界知识，无需任务特定的奖励工程。

局限：VLM 的判断可能不够精确；存在对抗样本问题。

5.3 基于成功检测器的奖励

训练一个二分类器判断当前状态是否”成功”，将分类概率作为奖励：

r(s)=P(success∣s)r(s) = P(\text{success} | s)r(s)=P(success∣s)

这比硬阈值的稀疏奖励更平滑，但需要标注成功/失败样本来训练检测器。

6. 奖励设计的常见陷阱

6.1 奖励黑客（Reward Hacking）

策略找到最大化奖励但不完成真实目标的捷径。

示例：奖励”减少物体到目标的距离”时，策略学到将物体猛推出桌面——物体虽然飞向目标方向但落到地上。

缓解方法：增加约束项（如保持物体在桌面上的奖励），或使用更全面的成功判定条件。

6.2 奖励尺度不平衡

不同奖励项的量级差异过大，导致策略只优化量级最大的项。

示例：距离奖励的量级为 [−0.5,0][-0.5, 0][−0.5,0]，成功奖励为 +100+100+100。策略学到尽快触发成功条件（即使方式不优雅），因为成功奖励远大于过程奖励。

缓解方法：归一化各奖励项的量级，或使用自适应权重。

6.3 奖励延迟

奖励信号与导致该奖励的动作之间存在时间延迟，使信用分配（Credit Assignment）困难。

7. 实践建议

场景推荐方案
简单抓取任务距离奖励 + 抓取成功奖励
多阶段操作任务分阶段奖励 + 势函数塑形
目标条件化策略稀疏奖励 + HER
缺乏先验知识逆强化学习或 VLM 奖励
Sim2Real 训练仿真中密集奖励 → 真机中稀疏验证

总结

奖励工程是强化学习从理论到实践的关键桥梁。稀疏奖励虽然无偏但难以学习，密集奖励加速了学习但引入了偏见和 Reward Hacking 风险，势函数塑形提供了理论上安全的加速方法，学习式奖励则试图消除人工设计的负担。实际项目中，往往需要组合使用多种策略，并通过反复迭代调优奖励函数。
