多模态 Tokenization 策略：视觉、语言与动作如何统一编码

Transformer 架构的核心抽象是序列到序列的映射：输入一个 Token 序列，输出一个 Token 序列。VLA（Vision-Language-Action）模型需要同时处理视觉图像、语言文本和机器人动作三种本质不同的信号。将这些异构信号统一编码为 Transformer 可处理的 Token 序列，是 VLA 架构设计的第一个关键决策。

本文分析各模态的 Tokenization 策略、统一编码的设计模式以及工程中的核心权衡。

1. 为什么需要统一编码？

Transformer 对输入的唯一要求是：一个由嵌入向量构成的序列。每个 Token 是一个固定维度的向量 xi∈Rd\mathbf{x}_i \in \mathbb{R}^dxi​∈Rd，模型通过自注意力机制在所有 Token 之间建立关联。

三种模态的原始表示完全不同：

模态原始形式维度
视觉RGB 图像张量H×W×3H \times W \times 3H×W×3
语言文本字符串变长字符序列
动作连续浮点向量DDD（如 7 维关节角度）

Tokenization 的目标是将三者映射到统一的嵌入空间 Rd\mathbb{R}^dRd，使 Transformer 能通过注意力机制学习跨模态的对齐关系。

2. 视觉 Tokenization

2.1 ViT Patch Embedding

Vision Transformer（ViT）将图像分割为固定大小的 Patch（如 16×1616 \times 1616×16 像素），每个 Patch 通过线性投影映射为一个 Token：

zi=Wp⋅flatten(patchi)+bp\mathbf{z}_i = \mathbf{W}_p \cdot \text{flatten}(\text{patch}_i) + \mathbf{b}_pzi​=Wp​⋅flatten(patchi​)+bp​

一张 224×224224 \times 224224×224 的图像在 Patch Size = 16 时产生 (224/16)2=196(224/16)^2 = 196(224/16)2=196 个 Token。

优势：简洁、高效，预训练 ViT 模型直接提供高质量视觉特征。

局限：Token 数量与图像分辨率的平方成正比。448×448448 \times 448448×448 分辨率产生 784 个 Token，显著增加注意力计算量。

2.2 双编码器融合

OpenVLA 等模型使用两个互补的视觉编码器：

  • SigLIP：侧重语义理解（“这是一个杯子”）。
  • DinoV2：侧重空间结构（物体的形状、位置、边界）。

两个编码器产生的 Token 序列拼接后输入 Transformer：

visual_tokens=[SigLIP(img);DinoV2(img)]\text{visual\_tokens} = [\text{SigLIP}(\text{img}); \text{DinoV2}(\text{img})]visual_tokens=[SigLIP(img);DinoV2(img)]

2.3 感知器重采样（Perceiver Resampler）

Flamingo 等模型使用 Perceiver Resampler 将可变数量的视觉 Token 压缩为固定数量的潜在 Token（Latent Tokens），降低后续 Transformer 的计算负担：

latent_tokens=CrossAttention(learnable_queries,visual_tokens)\text{latent\_tokens} = \text{CrossAttention}(\text{learnable\_queries}, \text{visual\_tokens})latent_tokens=CrossAttention(learnable_queries,visual_tokens)

这种方法在视觉 Token 数量过多时尤为有效。

3. 语言 Tokenization

3.1 BPE / SentencePiece

语言模态使用成熟的子词分词器（Subword Tokenizer），如 BPE（Byte Pair Encoding）或 SentencePiece：

"pick up the red cup" → [pick, ▁up, ▁the, ▁red, ▁cup]
                       → Token IDs: [5765, 891, 278, 2654, 6541]
                       → Embeddings: [e_1, e_2, e_3, e_4, e_5]

每个 Token ID 通过嵌入表（Embedding Table）映射为 ddd 维向量。

3.2 VLA 中的语言角色

在 VLA 模型中，语言 Token 通常扮演任务规范（Task Specification）的角色：

  • 指令式：“pick up the red cup and place it on the plate”
  • 目标描述式：“the cup is on the plate”

语言 Token 通过注意力机制与视觉 Token 交互，模型学习将语言描述的语义意图与视觉场景中的物体和位置关联起来。

4. 动作 Tokenization

动作模态的 Tokenization 是 VLA 设计中争议最大的部分，目前存在三种主流方案。

4.1 离散化 Binning

方法：将每个动作维度的连续值域均匀或非均匀地划分为 BBB 个 bin，将连续值映射为最近的 bin 索引：

token(ad)=round(ad−amin⁡amax⁡−amin⁡×(B−1))\text{token}(a_d) = \text{round}\left(\frac{a_d - a_{\min}}{a_{\max} - a_{\min}} \times (B - 1)\right)token(ad​)=round(amax​−amin​ad​−amin​​×(B−1))

每个 bin 索引通过嵌入表映射为向量，动作的 Tokenization 方式与语言完全一致。

代表模型：RT-2, OpenVLA

优势局限
与 LLM 框架完全兼容精度受 bin 数量限制（256 bin → ~0.4% 精度）
可直接使用 LLM 训练基础设施自回归生成逐维产生 Token，延迟高
交叉熵损失稳定不同维度的 bin 独立，忽略维度间相关性

4.2 连续嵌入

方法：使用线性投影或 MLP 将连续动作向量直接映射为嵌入向量，不经过离散化：

za=MLP(a)∈Rd\mathbf{z}_a = \text{MLP}(a) \in \mathbb{R}^dza​=MLP(a)∈Rd

代表模型：Octo, π0（动作专家的输入）

优势局限
无精度损失不能使用交叉熵损失，需要回归损失
保留维度间相关性需要专门的动作解码头
Token 数量可控与 LLM 预训练范式不兼容

4.3 VQ-VAE 离散化

方法：训练一个向量量化变分自编码器（VQ-VAE），将动作序列编码为离散码本（Codebook）中的索引序列：

a1:H→Encoderz1:K→Quantizez^1:K→Decodera^1:Ha_{1:H} \xrightarrow{\text{Encoder}} z_{1:K} \xrightarrow{\text{Quantize}} \hat{z}_{1:K} \xrightarrow{\text{Decoder}} \hat{a}_{1:H}a1:H​Encoder​z1:K​Quantize​z^1:K​Decoder​a^1:H​

优势：比简单 binning 更紧凑，码本可以学习到有意义的动作原语。

局限：需要额外训练 VQ-VAE，码本崩塌（Codebook Collapse）问题。

5. 统一序列的组装

三种模态的 Token 编码完成后，需要组装为一个统一序列输入 Transformer。常见的组装方式：

5.1 拼接式（Concatenation）

[CLS] [视觉 Token 1..N] [SEP] [语言 Token 1..M] [SEP] [动作 Token 1..K]

各模态 Token 简单拼接，通过位置编码和段编码（Segment Embedding）区分模态。

5.2 交叉注意力式（Cross-Attention）

不同模态的 Token 通过交叉注意力层交互，而非共享同一个自注意力序列。这种方式允许不同模态在各自的序列空间中独立处理，仅在交叉注意力层交换信息。

π0 采用这种方式：VLM Token 和动作 Token 在专门的 Action Expert 层中通过交叉注意力交互。

5.3 前缀式（Prefix）

视觉和语言 Token 作为前缀（Prefix），冻结不参与梯度更新。动作 Token 在前缀的条件下自回归生成或通过去噪生成。

6. 位置编码的设计

统一序列中，位置编码需要同时表达”模态内的位置”和”跨模态的顺序”：

策略描述
绝对位置编码全局位置 [0,1,2,…,N+M+K][0, 1, 2, \ldots, N+M+K][0,1,2,…,N+M+K]
分段位置编码每个模态内部独立编号，配合模态类型编码
2D 位置编码视觉 Token 使用 (row, col) 二维编码
旋转位置编码（RoPE）通过旋转矩阵编码相对位置，跨模态共享

7. Token 数量的工程影响

Transformer 的自注意力计算复杂度为 O(L2)O(L^2)O(L2)，LLL 为总 Token 数。各模态的 Token 数量直接影响推理速度和显存占用：

模态典型 Token 数优化手段
视觉（单相机）196-784Perceiver Resampler、降低分辨率
视觉（多相机）×2∼4\times 2 \sim 4×2∼4共享编码器、Token 池化
语言10-50通常不构成瓶颈
动作（离散化）H×DH \times DH×D（如 50×7=35050 \times 7 = 35050×7=350）减小 Chunk Size
动作（连续嵌入）HHH（如 50）固定 Chunk 长度

连续嵌入方式的 Token 数量显著少于离散化方式（HHH vs. H×DH \times DH×D），这是 π0 等模型选择连续嵌入的工程原因之一。

总结

多模态 Tokenization 的设计本质上是在表示精度、与预训练范式的兼容性以及计算效率三者之间的权衡。离散化 Token 与 LLM 范式天然兼容但损失精度；连续嵌入保留精度但需要专门的解码头；VQ-VAE 试图在两者之间找到平衡点。理解这些权衡，是设计和改进 VLA 架构的基础。
