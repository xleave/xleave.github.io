Action Chunking 机制解析：为什么 VLA 模型输出动作序列而非单步指令

在 VLA（Vision-Language-Action）模型和机器人策略学习的文献中，“Action Chunking”是一个反复出现的核心设计选择。它指的是策略模型在单次推理中输出未来多个时间步的动作序列（Action Chunk），而非每个时间步独立推理一个动作。

这一设计看似简单，却对系统的推理延迟容忍度、动作平滑性和任务成功率产生深远影响。本文解析 Action Chunking 的动机、实现方式以及工程中的关键考量。

1. 问题背景：单步推理的困境

1.1 传统范式

最直觉的策略执行方式是：每个控制周期调用一次策略模型，输入当前观测，输出一个动作，发送给机械臂执行。

for each timestep t:
    observation_t ← read_sensors()
    action_t ← policy(observation_t)
    robot.execute(action_t)

1.2 问题所在

当策略模型是一个大型 VLA 模型时，单次推理的延迟可能达到 100ms ~ 500ms。而典型的机械臂控制频率要求为：

控制模式频率单步时限
Franka FCI 底层1 kHz1 ms
笛卡尔阻抗控制100-200 Hz5-10 ms
上层策略控制5-20 Hz50-200 ms

即使在上层策略控制频率（5-20 Hz）下，VLA 模型的推理延迟也可能超过时限。单步推理范式要求每一步都在时限内完成，推理延迟构成硬瓶颈。

2. Action Chunking 的核心思想

Action Chunking 将策略的输出从单个动作扩展为一个动作序列：

observation_t → policy → [action_t, action_{t+1}, ..., action_{t+H-1}]

其中 HHH 是 Chunk Size（块大小），即一次推理覆盖的时间步数。

关键改变：推理频率从控制频率解耦。即使推理延迟较高，只要在当前 Chunk 执行完毕前完成下一次推理，系统就能持续运行。

for each chunk:
    observation_t ← read_sensors()
    action_chunk ← policy(observation_t)  // [a_t, a_{t+1}, ..., a_{t+H-1}]
    for i in range(H):
        robot.execute(action_chunk[i])
        // 可并行执行下一次推理

3. 为什么 Action Chunking 有效？

3.1 摊销推理延迟

假设推理延迟为 200ms，控制频率为 10 Hz（100ms/步）：

  • 单步推理：每 100ms 需要一次推理，但推理需要 200ms → 系统无法实时运行。
  • Chunk Size = 4：每 400ms 需要一次推理，推理延迟 200ms < 400ms → 系统可以实时运行，且有 200ms 的余量用于异步预取。

3.2 时间相关性建模

动作序列天然具有时间相关性。单步策略独立生成每一步动作，忽略了步与步之间的连贯性。Action Chunking 使模型在生成时就建模整段轨迹的时序结构，产生更平滑、更协调的运动。

3.3 缓解因果混淆

ACT（Action Chunking with Transformers）的论文指出，行为克隆（Behavioral Cloning）中存在因果混淆（Causal Confusion）：当多个专家演示在同一状态下采取不同动作时（多模态行为），单步策略容易学到各动作的平均值，导致无效行为。

Action Chunking 通过输出完整的动作序列，允许模型承诺（commit to）一条完整的行为轨迹，而非在每一步犹豫不决。

4. 实现方式

4.1 直接回归

最简单的方式是将策略的输出维度扩展为 H×DH \times DH×D，其中 DDD 是单步动作维度：

# 单步策略
action = policy(observation)  # shape: (D,)

# Action Chunking 策略
action_chunk = policy(observation)  # shape: (H, D)

ACT（Zhao et al., 2023）使用 CVAE（Conditional Variational Autoencoder）+ Transformer Decoder 实现这一方式。

4.2 Diffusion / Flow Matching

Diffusion Policy 和 π0 等模型使用去噪过程生成 Action Chunk：

噪声动作序列 z ~ N(0, I)  →  K 步去噪  →  clean action chunk
                shape: (H, D)                shape: (H, D)

这种方式天然支持多模态动作分布，适合存在多种合理行为的场景。

4.3 自回归 Token 生成

RT-2 等模型将每个动作维度编码为一个 Token，HHH 步动作产生 H×DH \times DH×D 个 Token，逐个自回归生成。缺点是 Token 数量与 Chunk Size 线性增长，推理延迟也线性增长。

5. Chunk 之间的衔接策略

当连续的 Action Chunk 之间存在重叠时，如何处理衔接是一个关键工程问题。

5.1 无重叠执行

最简单的方式：执行完当前 Chunk 的全部 HHH 步后，获取新观测并推理下一个 Chunk。

问题：Chunk 边界处可能出现不连续跳变。

5.2 指数加权平均（Temporal Ensemble）

ACT 提出的方法：允许连续 Chunk 之间存在重叠，对重叠区间内的动作使用指数加权平均进行融合：

at=∑kwk⋅at(k)∑kwka_t = \frac{\sum_{k} w_k \cdot a_t^{(k)}}{\sum_{k} w_k}at​=∑k​wk​∑k​wk​⋅at(k)​​

其中 at(k)a_t^{(k)}at(k)​ 是第 kkk 个 Chunk 对时刻 ttt 的预测，wk=exp⁡(−m⋅k)w_k = \exp(-m \cdot k)wk​=exp(−m⋅k) 是指数衰减权重，越新的 Chunk 权重越大。

5.3 滑动窗口重新规划

每推理一次获得新 Chunk，但只执行其中的前 KKK 步（K<HK < HK<H），然后基于新观测重新推理。这种”重新规划”策略平衡了响应性和稳定性。

6. Chunk Size 的选择

Chunk Size HHH 是一个关键超参数，其选择涉及多维权衡：

HHH 过小HHH 过大
推理延迟摊销不足对环境变化的响应滞后
时间相关性建模不足预测远期动作的误差累积
更接近单步策略更依赖开环执行

常见的 Chunk Size 选择：

模型 / 方法控制频率Chunk Size覆盖时长
ACT50 Hz1002s
Diffusion Policy10 Hz161.6s
π050 Hz501s

一般而言，Chunk Size 应覆盖 0.5s ~ 2s 的动作时域，具体取决于任务的动态性和推理延迟。

7. 与分层控制的关系

Action Chunking 可以视为一种轻量级的分层控制架构：

┌─────────────────────────────────────┐
│  高层：VLA 模型（低频，如 2-10 Hz）    │
│  输出：Action Chunk (H 步动作序列)     │
└────────────────┬────────────────────┘
                 ↓
┌─────────────────────────────────────┐
│  低层：插值器 + 控制器（高频，1 kHz）   │
│  输入：Action Chunk → 插值为连续指令    │
│  输出：关节力矩/位置指令               │
└─────────────────────────────────────┘

高层策略以低频率输出粗粒度的动作路标（Waypoints），低层控制器负责在路标之间进行平滑插值和安全约束检查。这种解耦使 VLA 模型不需要以控制频率运行。

总结

Action Chunking 的本质是将策略推理频率从底层控制频率中解耦出来。它通过一次推理覆盖多个时间步，同时解决了推理延迟过高、动作时序连贯性和多模态行为建模三个关键问题。理解 Chunk Size 的选择逻辑和 Chunk 间的衔接策略，是将 VLA 模型成功部署到真实机器人上的关键工程环节。
