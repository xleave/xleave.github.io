开放词汇物体检测在机器人抓取中的应用与局限

#物体检测#开放词汇#机器人抓取#CLIP#具身智能

传统的物体检测模型（如 YOLO、Faster R-CNN）在训练时定义固定的类别集合，只能检测已见过的物体类别。当机器人需要在开放世界中根据自然语言指令（如”递给我那个蓝色的马克杯”）抓取物体时，封闭类别集的检测模型无法满足需求。

开放词汇物体检测（Open-Vocabulary Object Detection, OVD）通过结合视觉-语言预训练模型，使检测器能够识别训练时未见过的物体类别。本文分析 OVD 在机器人抓取中的应用现状与技术局限。

1. 从封闭词汇到开放词汇

1.1 封闭词汇检测的局限

传统检测模型的类别集在训练时固定。例如，COCO 数据集定义了 80 个类别。模型无法检测第 81 个类别的物体，即使该物体在图像中清晰可见。

在机器人操作场景中，这意味着：

  • 每增加一种目标物体，都需要收集标注数据并重新训练。
  • 无法处理用户使用的非标准类别名称（如”那个东西”、“红色的那个”）。
  • 环境中的新物体完全不可见。

1.2 开放词汇检测的范式

OVD 模型接受文本查询作为输入，输出与查询语义匹配的目标框：

输入: 图像 + 文本查询 "blue mug"
输出: [bounding_box, confidence_score]

核心能力是将文本描述和视觉区域映射到共享的语义空间，通过计算相似度实现检测。

2. 技术方案

2.1 基于 CLIP 的方案

CLIP（Contrastive Language-Image Pretraining）通过对比学习将图像和文本编码到同一个嵌入空间。基于 CLIP 的 OVD 方案通常包含：

  1. 区域提议（Region Proposal）：使用类别无关的区域提议网络（如 RPN 或 Selective Search）生成候选框。
  2. 区域编码：将每个候选框裁剪并输入 CLIP 图像编码器，得到区域嵌入。
  3. 文本匹配：将文本查询输入 CLIP 文本编码器，计算与区域嵌入的余弦相似度。
  4. 阈值过滤：保留相似度超过阈值的区域作为检测结果。

2.2 代表模型

模型方法特点
OWL-ViT将 CLIP 的图像编码器与检测头结合Google 出品，端到端训练
Grounding DINODINO 检测器 + 文本-视觉融合开源，精度较高
GLIP短语 Grounding + 检测统一支持短语级别的定位
OWLv2OWL-ViT 的改进版，自训练扩展数据泛化能力更强

2.3 分割扩展：SAM + OVD

Segment Anything Model（SAM）提供强大的零样本分割能力。与 OVD 结合的流程为：

文本查询 → OVD 模型 → 目标框
                        ↓
目标框作为 prompt → SAM → 精确分割掩码

分割掩码比边界框提供更精确的物体轮廓，对抓取姿态估计有直接帮助。

3. 在机器人抓取中的应用

3.1 语言指令驱动的抓取

用户通过自然语言指定目标物体，OVD 模型定位物体，抓取规划器生成抓取姿态：

用户指令: "pick up the red cup"
     ↓
OVD 检测 → 定位 "red cup" 的位置
     ↓
深度图 → 估计 3D 位置
     ↓
抓取规划 → 生成 6-DOF 抓取姿态
     ↓
执行抓取

3.2 场景理解与任务规划

OVD 可以用于构建场景的语义地图——检测场景中所有可识别的物体及其位置，为 LLM 任务规划器提供结构化的场景描述：

objects = ovd.detect(image, queries=["cup", "plate", "spoon", "fork", ...])
scene_description = format_scene(objects)
plan = llm.plan(task="set the table", scene=scene_description)

3.3 主动感知

当 OVD 对某个物体的检测置信度较低时，机器人可以主动调整视角或移动相机以获取更好的观测，提升检测可靠性。

4. 当前局限

4.1 细粒度区分能力不足

OVD 模型擅长类别级别的检测（“杯子” vs. “盘子”），但在细粒度属性区分上表现较弱：

查询预期行为实际表现
”the larger cup”选择较大的杯子可能选错——缺乏大小比较能力
”the cup behind the plate”选择被盘子遮挡的杯子空间关系理解有限
”my cup”选择特定人的杯子无法理解所有权概念

4.2 3D 定位精度

OVD 在 2D 图像空间中检测目标框，要转化为机器人可用的 3D 坐标还需要：

  • 深度信息（来自 RGB-D 传感器或深度估计模型）。
  • 相机内外参标定。
  • 从 2D 框中心到 3D 物体中心的映射（对非对称物体可能不准确）。

这一 2D → 3D 转换引入的误差可能导致抓取失败。

4.3 推理延迟

基于大型 VLM 的 OVD 模型推理延迟较高：

模型典型延迟
OWL-ViT (Base)50-100 ms
Grounding DINO (Swin-T)80-150 ms
OWL-ViT (Large)150-300 ms

对于需要实时检测的动态抓取场景（如传送带上移动的物体），延迟可能成为瓶颈。

4.4 遮挡与杂乱环境

在物体密集堆叠或严重遮挡的场景中，OVD 模型的召回率显著下降。被其他物体遮挡超过 50% 的目标物体很可能无法被检测到。

4.5 域间差距

OVD 模型通常在互联网图像上预训练，与机器人视角的图像存在域间差距：

  • 机器人相机的视角通常是俯视或侧视，与互联网图像的视角分布不同。
  • 工业场景中的物体外观可能与互联网图像差异较大。
  • 照明条件、背景杂乱度等因素影响检测精度。

5. 改进方向

方向方法效果
域适应微调用少量机器人视角数据微调 OVD 模型提升特定场景精度
3D OVD直接在 3D 点云或体素上执行开放词汇检测消除 2D→3D 映射误差
交互式检测检测失败时主动移动相机或询问用户提升鲁棒性
多帧融合聚合多个视角的检测结果提升遮挡场景的召回率
与触觉结合在视觉检测不确定时用触觉验证多模态互补

总结

开放词汇物体检测为机器人抓取提供了从封闭类别集到自然语言驱动物体定位的范式转变。它使机器人能够理解任意语言描述的目标物体，是构建通用操作系统的关键组件。然而，细粒度区分、3D 定位精度、推理延迟和遮挡鲁棒性等局限仍需持续改进。将 OVD 与深度传感、3D 推理和主动感知结合，是当前的主要发展方向。
