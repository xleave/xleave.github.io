具身智能中的观测空间设计：RGB、深度、点云与触觉的工程权衡

具身智能（Embodied AI）系统的策略模型从观测数据中提取信息并做出决策。观测空间（Observation Space）的设计——即选择哪些传感模态、如何表示和编码这些数据——直接影响策略学习的效率、泛化能力和部署难度。

本文分析机器人操作任务中常用传感模态的特性，讨论观测空间设计中的工程权衡。

1. 观测空间设计的核心问题

观测空间设计需要回答三个问题：

  1. 信息充分性：所选模态是否提供了完成任务所需的全部信息？
  2. 学习效率：策略模型能否从这些观测中高效地提取有用特征？
  3. 部署可行性：传感器的成本、可靠性和延迟是否满足实际部署要求？

2. RGB 图像

2.1 特点

RGB 图像是最常用的观测模态，提供丰富的视觉语义信息：物体类别、颜色、纹理、空间布局以及人类可理解的场景描述。

优势局限
信息密度高，包含语义和空间信息对光照变化敏感
预训练视觉模型（ViT、ResNet）可直接复用缺乏显式深度信息
传感器成本低、可靠性高遮挡场景中信息缺失
与 VLM/VLA 架构天然兼容高分辨率导致计算量大

2.2 相机配置

典型的机器人操作系统使用多相机配置：

  • 外部相机（Third-person View）：提供全局场景视角，适合高层任务理解。
  • 腕部相机（Wrist Camera）：安装在机械臂末端，提供操作目标的近距离视角，适合精密操作。
  • 多视角融合：两路或更多视角的图像共同输入策略模型，提供更完整的 3D 理解。

VLA 模型（如 π0）通常使用 1 路外部相机 + 1 路腕部相机的双视角配置。

3. 深度图（Depth Map）

3.1 特点

深度图记录每个像素到相机的距离，提供显式的 3D 几何信息。常见的深度传感器包括结构光（如 RealSense D435）、飞行时间（ToF）和立体视觉。

优势局限
提供显式的 3D 几何信息透明/反光物体的深度测量不准
对光照变化不敏感传感器量程和精度有限
可直接用于距离判断和碰撞检测缺乏语义和纹理信息
与 RGB 对齐后构成 RGB-D深度图通常有噪声和空洞

3.2 RGB-D 融合

RGB-D 是 RGB 与深度的逐像素对齐融合，是机器人操作中最常见的观测格式。两种模态互补：RGB 提供语义，深度提供几何。

融合策略包括：

  • Early Fusion：将深度作为第 4 个通道（RGBD），直接输入卷积网络。
  • Late Fusion：分别编码 RGB 和 Depth，在特征层拼接或融合。
  • Mid Fusion：在中间层通过交叉注意力融合。

4. 点云（Point Cloud）

4.1 特点

点云是三维空间中的离散点集合，每个点包含坐标 (x,y,z)(x, y, z)(x,y,z) 和可选的颜色、法向量等属性。点云可以从深度图反投影（Back-projection）得到，也可以从 LiDAR 直接获取。

优势局限
直接在 3D 空间中表示几何信息数据量大且不规则
不受视角变换影响（世界坐标系）需要专用网络架构（PointNet/PointNet++）
适合 6-DOF 抓取姿态估计点云拼接需要精确的相机标定
天然支持碰撞检测和运动规划实时处理的计算成本较高

4.2 常用网络架构

  • PointNet：对每个点独立提取特征，再通过全局池化聚合。对排列不变性有优雅的处理。
  • PointNet++：在 PointNet 基础上引入层次化的局部区域分组，捕获局部几何结构。
  • Point Transformer：将自注意力机制应用于点云，建模点与点之间的长程依赖。

4.3 适用场景

点云在以下场景中特别有价值：

  • 6-DOF 抓取：需要在 3D 空间中精确估计抓取姿态。
  • 堆叠/装配任务：需要精确的空间关系推理。
  • 多视角融合：多个相机的点云可以直接在世界坐标系中拼接。

5. 触觉传感（Tactile Sensing）

5.1 特点

触觉传感器安装在机器人的指尖或抓手上，感知接触力、接触区域形状和滑动信号。常见类型包括基于视觉的触觉传感器（如 GelSight、DIGIT）和基于电阻/电容的传感器阵列。

优势局限
提供接触层面的信息（力、形变）仅在接触时有信号
对视觉遮挡不敏感传感器成本和集成复杂度较高
可检测滑动、纹理、硬度数据表示和标注不标准化
对精密操作（如插拔）至关重要预训练模型生态不成熟

5.2 基于视觉的触觉传感

GelSight 系列传感器将触觉信息转化为视觉图像：弹性体表面的形变被内置相机捕捉，产生高分辨率的接触几何图像。这种设计使得视觉领域的预训练模型可以直接用于触觉特征提取。

6. 本体感觉（Proprioception）

本体感觉是机器人自身状态的感知，包括：

  • 关节角度（Joint Position）
  • 关节角速度（Joint Velocity）
  • 关节力矩（Joint Torque）
  • 末端执行器位姿（End-effector Pose）
  • 夹爪开合度（Gripper Width）

本体感觉是几乎所有策略模型的必要输入，它提供了动作空间的当前状态，使策略能计算”从当前状态到目标状态需要多大的变化”。

7. 观测空间设计的权衡矩阵

模态信息类型预训练生态传感成本实时性泛化能力
RGB语义 + 空间丰富（ViT, CLIP）低高强（VLM 先验）
深度几何中等中高中等
点云3D 几何有限（PointNet）中高中场景级泛化较好
触觉接触力/形变不成熟高高弱（传感器特定）
本体感觉机器人状态不适用无额外成本极高机器人形态特定

8. 设计建议

8.1 最小可行观测

对于大多数桌面操作任务，推荐的最小可行观测配置为：

观测 = 外部 RGB 图像 + 腕部 RGB 图像 + 本体感觉（关节状态 + 夹爪）

这一配置与主流 VLA 模型（π0、OpenVLA）兼容，且传感成本可控。

8.2 渐进式扩展

根据任务需求逐步引入更多模态：

  1. 基础：RGB + 本体感觉 → 覆盖大多数抓取和放置任务。
  2. +深度：RGB-D → 需要精确距离判断的任务（如堆叠）。
  3. +点云：6-DOF 抓取、非结构化环境。
  4. +触觉：精密插拔、柔性物体操作、力敏感任务。

8.3 观测预处理的一致性

无论选择哪种模态，确保训练和部署时的观测预处理一致：

  • 图像分辨率、裁剪和归一化参数。
  • 深度图的缺失值填充策略。
  • 点云的降采样密度和坐标系定义。
  • 本体感觉的归一化范围。

观测预处理的不一致是导致策略在 Sim2Real 迁移或跨场景部署时性能下降的常见原因。

总结

观测空间设计是具身智能系统工程中的第一个关键决策。不存在普适的最优配置——选择取决于任务需求、可用硬件和策略模型的能力。核心原则是：从最小可行观测出发，根据任务需求渐进式扩展，并始终确保训练与部署之间的预处理一致性。
