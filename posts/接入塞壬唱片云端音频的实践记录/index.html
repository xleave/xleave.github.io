接入塞壬唱片云端音频的实践记录

#音乐#前端#CORS#API
jie ru sai ren chang pian yun duan yin pin de shi jian ji lu jrsrcpydypdsjjl yin yue qian duan C O R S A P I yy qd CORS API ji shu js

起因

在为博客选择背景音乐时，一直在寻找一个既无需自己托管音频文件、又能保证版权合规的方案。起初曾考虑接入网易云音乐或 Bilibili，但两者均需要账号或较为复杂的 Token 鉴权流程，且网易云的外链限制较为严格。

后来想到明日方舟的官方音乐子品牌——塞壬唱片（Monster Siren Records，简称 MSR）。塞壬唱片面向公众提供了独立的音乐平台（monster-siren.hypergryph.com），其音频资源部署在鹰角自有 CDN（web.hycdn.cn）上，且接口无需任何鉴权即可访问。

API 结构分析

塞壬唱片对外暴露了一套简洁的 REST 风格接口，主要用到以下端点：

获取单曲详情

GET https://monster-siren.hypergryph.com/api/song/{cid}

响应示例（精简）：

{
  "code": 0,
  "msg": "OK",
  "data": {
    "cid": "880374",
    "name": "Speed of Light",
    "albumCid": "886702",
    "sourceUrl": "https://web.hycdn.cn/siren/audio/20210322/XXXXXX.mp3",
    "lyricUrl": null,
    "mvUrl": null
  }
}

cid 是每首曲目的唯一标识符，可以在官网的页面 URL 中直接获取。sourceUrl 是实际可播放的 MP3 地址，由鹰角 CDN 托管，无需二次鉴权。

获取专辑信息

GET https://monster-siren.hypergryph.com/api/album/{albumCid}/detail

可以进一步获取专辑封面、曲目列表等信息。专辑封面 URL 形如：

https://web.hycdn.cn/siren/pic/{date}/{hash}.png

CORS 跨域问题

直接在前端 JavaScript 中 fetch 上述接口，会遇到跨域（CORS）错误：

Access to fetch at 'https://monster-siren.hypergryph.com/api/song/...'
from origin 'https://yourdomain.com' has been blocked by CORS policy:
No 'Access-Control-Allow-Origin' header is present on the requested resource.

塞壬唱片的 API 服务器并未对外站开放 CORS 响应头，这是浏览器的同源安全策略造成的限制。

解决方案：使用公共 CORS 代理

最简单且无需服务端的方案是借助公共 CORS 代理服务。这类服务的原理是：代理服务器以服务端身份转发请求，并在响应中追加 Access-Control-Allow-Origin: * 头，从而绕过浏览器的同源限制。

这里选用了 corsproxy.io，使用方式极为简单，只需在目标 URL 前拼接代理前缀即可：

https://corsproxy.io/?{目标URL}

实际请求改写如下：

// 原始请求（受 CORS 限制，无法直接访问）
// const res = await fetch(`https://monster-siren.hypergryph.com/api/song/${cid}`);

// 通过 CORS 代理转发
const res = await fetch(
  `https://corsproxy.io/?https://monster-siren.hypergryph.com/api/song/${cid}`
);
const data = await res.json();

if (data.code === 0 && data.data?.sourceUrl) {
  audio.src = data.data.sourceUrl;
}

注意： 公共 CORS 代理服务不保证 SLA，在高并发或网络异常情况下可能出现延迟或失败。建议添加 try/catch 并配置本地备用 URL 作为 fallback。

播放器实现方案

本站音乐播放器基于原生 HTML <audio> 元素实现，采用全局单例 + 曲目列表的架构：

数据结构

曲目信息维护在 src/data/music.json 文件中，每个条目包含：

{
  "cid": "880374",
  "name": "Speed of Light",
  "artist": "塞壬唱片-MSR",
  "url": "",
  "cover": "https://web.hycdn.cn/siren/pic/..."
}
  • cid 用于通过 API 获取实时音频地址
  • url 作为后备地址（当 API 请求失败时使用）
  • cover 使用从官方 API 或专辑信息中获取的封面图 URL

加载逻辑

const loadGlobalSong = async (idx) => {
  const song = songs[idx];

  if (song.cid) {
    try {
      const res = await fetch(
        `https://corsproxy.io/?https://monster-siren.hypergryph.com/api/song/${song.cid}`
      );
      const data = await res.json();
      if (data.code === 0 && data.data?.sourceUrl) {
        audio.src = data.data.sourceUrl;
      } else {
        audio.src = song.url; // fallback
      }
    } catch (e) {
      console.error("Failed to fetch MSR song URL:", e);
      audio.src = song.url; // fallback
    }
  } else {
    audio.src = song.url;
  }

  if (isPlaying) {
    audio.play().catch(console.error);
  }
};

跨页面持续播放

Astro 使用 <ViewTransitions> 实现页面切换动画，这会触发 DOM 的局部替换，可能中断音频播放。解决方式是在全局 <audio> 元素上监听 astro:after-swap 事件，在页面切换后重新挂载播放器事件监听器，而不重新创建 <audio> 元素本身。

document.addEventListener("astro:after-swap", () => {
  setupMiniPlayer(); // 重新绑定 DOM 引用与事件
});

版权说明

塞壬唱片所有曲目版权归上海鹰角网络科技有限公司所有。本方案仅通过官方公开接口获取资源链接进行个人网站的非商业用途播放，不涉及音频文件的存储、二次分发或盈利行为。如有版权异议，应及时停止使用。

在使用时建议：

  • 仅用于非商业的个人博客或展示页面
  • 在页面明确标注音乐来源（塞壬唱片 / Monster Siren Records）
  • 不对音频文件进行本地缓存或镜像托管

小结

方案要素具体实现
音频来源塞壬唱片官方 CDN（无需鉴权）
CORS 处理corsproxy.io 公共代理
播放器架构原生 <audio> 单例 + 曲目 JSON
跨页面持续播放Astro astro:after-swap 事件重挂载
失败降级本地 url 字段 fallback

整体方案的实现代价较低，无需后端服务，适合静态托管的个人博客。唯一的潜在风险在于依赖第三方 CORS 代理的稳定性，后续可考虑自建轻量代理服务器（如 Cloudflare Worker）来替代。
