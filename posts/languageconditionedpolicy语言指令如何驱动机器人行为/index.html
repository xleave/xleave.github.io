Language-Conditioned Policy：语言指令如何驱动机器人行为

语言条件化策略（Language-Conditioned Policy）是具身智能中的核心范式：策略模型不再硬编码单一任务，而是以自然语言指令作为条件输入，根据指令的语义内容动态调整行为。这使得单一策略模型可以执行开放词汇的多种任务。

本文分析语言指令如何作为条件信号融入策略模型，讨论关键的技术设计与泛化挑战。

1. 为什么用语言作为任务规范？

在传统的多任务强化学习中，任务通常通过以下方式指定：

方式描述局限
任务 ID（One-hot）用固定编号标识任务无法泛化到新任务
目标状态提供目标图像或目标状态向量需要预先获取目标信息
奖励函数每个任务对应不同的奖励无法在推理时动态切换

语言指令的优势在于：

  • 开放性：可以描述训练时未见过的任务组合（如”把红色杯子放到蓝色盘子左边”）。
  • 人类自然接口：用户无需了解机器人的技术细节，直接用自然语言交互。
  • 组合泛化：语言的组合结构（名词 + 动词 + 属性 + 空间关系）天然支持任务的组合泛化。

2. 语言嵌入方法

2.1 预训练语言模型

最常用的方法是利用预训练语言模型将指令编码为固定维度的嵌入向量：

# 使用 CLIP 文本编码器
text_embedding = clip.encode_text("pick up the red cup")
# shape: (512,)

# 使用 Sentence-BERT
text_embedding = sbert.encode("pick up the red cup")
# shape: (384,)

# 使用 LLM 的最后隐藏层
tokens = tokenizer("pick up the red cup")
hidden_states = llm(tokens).last_hidden_state
text_embedding = hidden_states.mean(dim=1)  # 平均池化

2.2 嵌入方法对比

方法维度语义质量视觉对齐计算成本
CLIP Text Encoder512高强（与视觉预训练对齐）低
Sentence-BERT384/768高弱低
LLM Hidden State4096+极高取决于 LLM高
随机初始化嵌入表自定义低（需大量训练数据）无极低

在 VLA 模型中，CLIP 文本编码器因其与视觉模态的预训练对齐而最为常用。

2.3 Token 级别 vs. 句子级别

  • 句子级别嵌入：将整条指令编码为单个向量。信息损失可能较大，但使用简便。
  • Token 级别嵌入：保留指令中每个 Token 的嵌入，形成 Token 序列。信息更丰富，支持通过注意力机制关注指令的特定部分。

现代 VLA 模型（如 RT-2、π0）普遍使用 Token 级别嵌入，与 Transformer 的序列输入格式天然兼容。

3. 条件化架构

将语言嵌入融入策略网络的方式有多种：

3.1 拼接条件化（Concatenation）

将语言嵌入与其他输入（如视觉特征、本体感觉）拼接后输入 MLP：

combined = torch.cat([visual_features, language_embedding, proprio])
action = mlp(combined)

优缺点：实现简单，但语言与视觉的交互仅通过后续网络层隐式实现。

3.2 FiLM 条件化

Feature-wise Linear Modulation（FiLM）使用语言嵌入生成缩放和偏移参数，对视觉特征进行调制：

γ,β=MLP(language_embedding)\gamma, \beta = \text{MLP}(\text{language\_embedding})γ,β=MLP(language_embedding) output=γ⊙visual_features+β\text{output} = \gamma \odot \text{visual\_features} + \betaoutput=γ⊙visual_features+β

优缺点：语言嵌入直接调制视觉特征的激活，融合更紧密。在 CNN Backbone 中尤为有效。

3.3 交叉注意力条件化（Cross-Attention）

语言 Token 作为 Key 和 Value，视觉 Token 作为 Query，通过交叉注意力层使视觉特征关注指令中的相关部分：

Attention(Qvisual,Klanguage,Vlanguage)\text{Attention}(Q_{\text{visual}}, K_{\text{language}}, V_{\text{language}})Attention(Qvisual​,Klanguage​,Vlanguage​)

优缺点：最灵活的融合方式，允许视觉特征选择性地关注指令中的特定词汇（如”红色""左边”）。是 Transformer 架构的标准选择。

3.4 前缀条件化（Prefix Conditioning）

将语言 Token 作为 Transformer 输入序列的前缀，与视觉 Token 拼接后共同参与自注意力：

输入序列: [语言 Token 1..M] [SEP] [视觉 Token 1..N] [动作 Token 1..K]

RT-2 和 OpenVLA 采用这种方式，将语言指令视为与图像描述类似的文本输入。

4. 语言指令的粒度

语言指令的粒度（Granularity）显著影响策略的行为和泛化能力：

4.1 粒度谱系

粒度示例策略需要推理的内容
高层目标”clean the table”整个任务规划 + 所有子动作
子任务”pick up the cup”抓取的完整动作序列
动作描述”move the arm to the left”单步运动方向和幅度
精确指令”rotate wrist 30 degrees clockwise”精确的参数化动作

4.2 粒度与泛化的关系

高层指令要求策略模型具备更强的推理能力和更长的时域规划，但提供了更大的行为灵活性。低层指令减少了歧义但限制了策略的自主决策空间。

实践中的折中方案是分层架构：

高层 LLM 规划器: "clean the table" → ["pick up cup", "place in sink", ...]
                                          ↓
低层 VLA 策略:   "pick up cup" → 连续动作序列

5. 泛化挑战

5.1 组合泛化

训练数据中包含”pick up red cup”和”pick up blue plate”，策略能否泛化到”pick up blue cup”？

这要求语言编码器能将”blue”和”cup”的语义分解为独立的属性和对象概念，而非记忆指令的整体模式。

预训练语言模型（尤其是 CLIP）在组合泛化上表现较好，因为它们在互联网规模的数据上学到了属性-对象的解耦表示。

5.2 指令的同义改写

“pick up the cup”、“grab the mug”、“take the cup” 应当触发相同的行为。预训练语言模型将这些同义表达映射到嵌入空间中的相近位置，为同义泛化提供了基础。

5.3 否定和条件指令

“don’t touch the red one”、“pick up the cup only if it’s empty” 等包含否定或条件逻辑的指令，对当前的策略模型来说仍然具有很大挑战。这需要更深层的语义理解，超越了简单的语义相似度匹配。

5.4 指代消解

“pick up that one”、“move it to the left” 中的代词需要结合上下文（视觉场景 + 对话历史）进行消解。多轮对话场景对策略模型的上下文理解能力提出了更高要求。

6. 评估方法

6.1 指令跟随准确率

给定一条指令和一次执行轨迹，判断策略是否正确完成了指令描述的任务。通常使用成功率（Success Rate）作为主要指标。

6.2 语言泛化测试

使用训练时未见过的指令措辞测试策略：

测试维度示例
同义改写”pick up” → “grab”, “take”
未见属性组合”green cup”（训练中只有 red/blue cup）
未见空间关系”behind the plate”（训练中只有 on/next to）
未见物体类别”pick up the stapler”（训练中无 stapler）

总结

语言条件化策略将自然语言从人机交互界面提升为策略模型的核心控制信号。通过预训练语言模型的语义编码和交叉注意力条件化架构，策略模型能够理解和执行开放词汇的多样化指令。然而，组合泛化、否定逻辑和指代消解等深层语义理解挑战仍是当前的瓶颈，需要更强大的多模态推理能力来突破。
