Teacher-Student 蒸馏框架在 Sim2Real 策略迁移中的应用

Teacher-Student 蒸馏框架是 Sim-to-Real 策略迁移中的一种有效方法。其核心思想是：在仿真中利用真实世界不可获得的特权信息（Privileged Information）训练一个高性能的 Teacher 策略，然后通过知识蒸馏将 Teacher 的行为迁移到仅使用真实世界可获得观测的 Student 策略上。

1. 动机

1.1 特权信息的价值

在仿真中，可以获取真实世界中无法直接测量的信息：

特权信息仿真中可获取真实世界中是否可获取
物体的精确质量✓✗（需要称量）
物体的精确位姿✓（GT pose）✗（需要估计）
精确的接触力✓部分（力传感器有噪声）
地形高度图✓✗（需要扫描）
摩擦系数✓✗
环境参数✓✗

使用这些特权信息训练的策略通常比仅使用真实世界可获取观测训练的策略性能高出很多。

1.2 部署约束

Teacher 策略虽然性能高，但由于依赖特权信息而无法直接部署到真机。Student 策略仅使用可部署的观测（如 RGB 图像、本体感觉），可以直接部署。

2. 两阶段训练框架

2.1 阶段一：训练 Teacher

Teacher 策略 π_T:
  输入: 可部署观测 o + 特权信息 p
  训练方法: RL（PPO/SAC）
  目标: 最大化任务回报

Teacher 使用完整的状态信息和特权信息，在仿真中通过强化学习训练到高性能。由于信息充分，Teacher 通常能快速学到高质量的策略。

2.2 阶段二：蒸馏 Student

Student 策略 π_S:
  输入: 仅可部署观测 o
  训练方法: 监督学习（模仿 Teacher 的行为）
  目标: 最小化与 Teacher 动作的差距

Student 通过模仿 Teacher 的行为来学习，而非直接通过 RL 探索。这避免了从零开始 RL 训练的探索困难。

2.3 蒸馏损失

Ldistill=Eo∼D[∥πS(o)−πT(o,p)∥2]\mathcal{L}_{\text{distill}} = \mathbb{E}_{o \sim \mathcal{D}} \left[ \| \pi_S(o) - \pi_T(o, p) \|^2 \right]Ldistill​=Eo∼D​[∥πS​(o)−πT​(o,p)∥2]

或使用 KL 散度匹配分布（适用于随机策略）：

LKL=Eo∼D[DKL(πT(⋅∣o,p)∥πS(⋅∣o))]\mathcal{L}_{\text{KL}} = \mathbb{E}_{o \sim \mathcal{D}} \left[ D_{\text{KL}}(\pi_T(\cdot|o, p) \| \pi_S(\cdot|o)) \right]LKL​=Eo∼D​[DKL​(πT​(⋅∣o,p)∥πS​(⋅∣o))]

3. Teacher 的特权信息设计

3.1 运动控制任务（Locomotion）

ETH Zurich 的四足机器人（ANYmal）工作中，典型的特权信息包括：

  • 脚下地形的高度图（地形特征）
  • 当前的域随机化参数（质量、摩擦、电机强度）
  • 精确的脚-地接触状态

Student 仅使用关节角度、角速度和 IMU 数据。

3.2 操作任务（Manipulation）

在灵巧手操作任务中：

  • Teacher 使用物体的精确 6D 位姿和速度。
  • Student 使用指尖触觉传感器和本体感觉。

3.3 视觉策略

  • Teacher 使用物体的 Ground Truth 状态向量。
  • Student 使用 RGB 图像，需要通过视觉编码器隐式推断状态。

4. Student 架构设计

4.1 状态估计模块

Student 通常包含一个隐式或显式的状态估计模块，从可部署观测中推断 Teacher 所依赖的特权信息：

可部署观测 o → 编码器 → 潜在表征 ẑ ≈ 特权信息 p 的充分统计量
                              ↓
                         策略头 → 动作 a

4.2 历史编码器

某些特权信息（如摩擦系数）无法从单帧观测中推断，但可以从观测的历史序列中隐式推断（通过观察机器人与环境交互的模式）。

常用架构：

  • RNN/LSTM：编码变长历史序列。
  • 1D 卷积（TCN）：对固定长度的历史窗口进行卷积。
  • Transformer：对历史 Token 序列使用自注意力。

4.3 联合蒸馏 + RL

纯蒸馏可能使 Student 的性能上限被 Teacher 限制。一种改进是在蒸馏的基础上加入 RL 损失：

L=λdistillLdistill+λRLLRL\mathcal{L} = \lambda_{\text{distill}} \mathcal{L}_{\text{distill}} + \lambda_{\text{RL}} \mathcal{L}_{\text{RL}}L=λdistill​Ldistill​+λRL​LRL​

RL 损失允许 Student 在 Teacher 行为的基础上进一步优化，可能超越 Teacher 的性能（因为 Student 学到了适应自身感知能力的策略）。

5. DAgger 式在线蒸馏

5.1 离线蒸馏的问题

标准的蒸馏使用 Teacher 采集的轨迹数据训练 Student。但由于 Student 的观测空间不同，其执行轨迹可能偏离 Teacher 的数据分布，导致分布偏移（Distribution Shift）。

5.2 DAgger 改进

DAgger（Dataset Aggregation）通过迭代式数据收集缓解分布偏移：

for each iteration:
    1. Student 在仿真中执行策略，产生轨迹
    2. 在 Student 访问过的状态上查询 Teacher 的动作
    3. 将 (Student 状态, Teacher 动作) 加入训练集
    4. 在聚合数据集上重新训练 Student

这确保 Student 在自己可能访问的状态分布上都有 Teacher 的指导。

6. 应用案例

6.1 OpenAI 灵巧手解魔方

  • Teacher：使用 RL + 域随机化训练，输入包含魔方的精确状态。
  • Student：仅使用手指关节角度和指尖触觉，通过蒸馏学习。
  • 域随机化范围：通过 ADR 自动扩展。

6.2 ANYmal 四足机器人

  • Teacher：使用精确地形高度图 + 域随机化参数。
  • Student：使用 IMU + 关节编码器 + 历史编码器（TCN）。
  • 部署：Student 直接部署到真机，无需真实数据微调。

6.3 灵巧手 Allegro 操作

  • Teacher：使用物体精确位姿。
  • Student：使用指尖触觉图像（GelSight）。
  • 视觉编码：触觉图像通过 CNN 编码为潜在向量。

7. 局限与挑战

局限说明
Teacher 质量上限Student 的性能通常受限于 Teacher 的策略质量
信息瓶颈Student 可能无法从有限观测中推断所有必要信息
蒸馏效率大型 Teacher（如 VLA 模型）的蒸馏计算成本高
行为模式坍缩蒸馏过程中可能丢失 Teacher 的多模态行为
仿真依赖Teacher 仍需在仿真中训练，Gap 仍然存在

总结

Teacher-Student 蒸馏框架将 Sim2Real 迁移分解为两个更容易的子问题：（1）在信息充分的仿真中训练高性能策略；（2）将该策略的知识迁移到可部署的感知空间。这种分解避免了直接从有限观测中进行 RL 探索的困难，同时通过 DAgger 和联合 RL 等技术缓解蒸馏过程中的分布偏移和信息损失。该框架已在四足机器人、灵巧手操作等场景中证明了其有效性。
