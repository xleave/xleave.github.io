域随机化（Domain Randomization）的原理、参数设计与局限性

#域随机化#Sim2Real#仿真#泛化#强化学习

域随机化（Domain Randomization, DR）是 Sim-to-Real 迁移中最广泛使用的缓解策略之一。其核心假设是：如果策略在足够多样化的仿真环境中都能成功，那么真实环境只是这个多样化空间中的一个实例，策略自然能泛化到真实世界。

1. 基本原理

1.1 形式化描述

设仿真环境参数为 ξ∈Ξ\xi \in \Xiξ∈Ξ，策略 πθ\pi_\thetaπθ​ 在参数为 ξ\xiξ 的环境中的期望回报为 J(θ,ξ)J(\theta, \xi)J(θ,ξ)。标准训练优化单一仿真参数下的性能：

θ∗=arg⁡max⁡θJ(θ,ξ0)\theta^* = \arg\max_\theta J(\theta, \xi_0)θ∗=argmaxθ​J(θ,ξ0​)

域随机化将目标改为在参数分布 p(ξ)p(\xi)p(ξ) 上的期望性能：

θ∗=arg⁡max⁡θEξ∼p(ξ)[J(θ,ξ)]\theta^* = \arg\max_\theta \mathbb{E}_{\xi \sim p(\xi)} [J(\theta, \xi)]θ∗=argmaxθ​Eξ∼p(ξ)​[J(θ,ξ)]

这迫使策略在各种参数配置下都表现良好，而非过拟合到特定参数。

1.2 直觉

域随机化可以类比为数据增强（Data Augmentation）在策略学习中的推广：

  • 图像分类中的数据增强：随机裁剪、旋转、颜色抖动 → 分类器对这些变换不变。
  • 域随机化：随机物理参数、光照、材质 → 策略对这些参数不变。

2.1 物理参数随机化

  • 深度图噪声和缺失区域。
  • 关节编码器偏移和噪声。
  • 力传感器漂移和量化误差。

3. 参数范围设计

3.1 过窄 vs. 过宽

参数范围的选择是 DR 的核心设计决策：

范围过窄范围过宽
真实参数可能超出范围 → 迁移失败策略需要在过大的参数空间中泛化 → 性能下降
策略过拟合到窄范围策略过于保守，避免在极端参数下失败的行为

3.2 设计原则

  1. 以真实参数为中心：如果能粗略估计真实参数值，以该值为中心设置随机化范围。
  2. 包含安全余量：范围应大于估计不确定性，留出 20%−50%20\%-50\%20%−50% 的安全余量。
  3. 逐维度调优：不同参数对策略性能的敏感度不同。可以通过消融实验确定各参数的合理范围。

4. 自适应域随机化（ADR）

4.1 动机

手动设计随机化范围费时且可能不优。自适应域随机化（Adaptive Domain Randomization, ADR）自动调整随机化范围。

4.2 AutoDR / ADR 方法

OpenAI 在解魔方手（Rubik’s Cube）项目中使用的 ADR 方法：

  1. 初始化每个参数的随机化范围为较窄的区间。
  2. 在训练过程中，监控策略在当前参数范围内的成功率。
  3. 如果成功率超过阈值，逐步扩大随机化范围。
  4. 如果成功率低于阈值，缩小随机化范围。
for each parameter p:
    if success_rate > threshold_high:
        range_p = range_p * (1 + expansion_rate)
    elif success_rate < threshold_low:
        range_p = range_p * (1 - shrink_rate)

优势：自动找到策略能承受的最大随机化范围，无需人工调优。

4.3 BayesSim

BayesSim 使用贝叶斯推断，根据少量真实数据估计仿真参数的后验分布，然后将该后验分布作为域随机化的采样分布。

5. 高级技巧

5.1 课程式随机化

训练初期使用较窄的随机化范围（或不随机化），让策略先学到基本行为；训练后期逐步扩大范围，使策略逐步泛化。

5.2 分层随机化

不同层级的策略使用不同维度的随机化：

  • 低层控制策略：随机化物理参数（质量、摩擦、延迟）。
  • 高层任务策略：随机化视觉参数（光照、纹理、相机位姿）。

5.3 对抗式随机化

使用对抗网络自动生成使策略失败的参数配置，然后在这些困难配置上重点训练。这比均匀随机采样更高效。

6. 局限性

6.1 保守策略问题

过大的随机化范围会导致策略学到过于保守的行为——在所有参数配置下都”不太差”但也”不够好”。这在需要激进动作的任务中（如快速抓取移动物体）尤为明显。

6.2 不可随机化的 Gap

某些 Gap 无法通过参数随机化来覆盖：

  • 仿真器根本不支持的物理现象（如布料的微观褶皱）。
  • 真实传感器特有的伪影（如 RealSense 的飞行像素）。
  • 真实环境中的动态干扰（如人的触碰）。

6.3 维度灾难

当随机化参数维度过高时，均匀采样覆盖参数空间所需的样本量呈指数增长。100 个参数各取 10 个值需要 1010010^{100}10100 种组合——远超可行计算量。

6.4 参数相关性

真实世界中，物理参数之间往往存在相关性（如质量和惯量矩阵不独立）。独立随机化各参数可能产生物理上不可能的组合，浪费训练样本。

7. 与其他方法的组合

DR 很少单独使用。实际项目中常见的组合：

组合方式效果
DR + 系统辨识以辨识参数为中心，DR 覆盖辨识误差
DR + 域适应DR 粗略泛化，域适应精确对齐
DR + 真实数据微调DR 提供鲁棒初始化，微调消除残余 Gap
DR + Teacher-StudentTeacher 用 DR 训练，Student 蒸馏为紧凑策略

总结

域随机化的核心价值在于：用计算资源换取真实数据。通过在仿真中大量采样多样化的环境参数，策略被迫学到对参数变化鲁棒的行为，从而能泛化到真实世界。但其有效性依赖于随机化范围的合理设计和仿真器对关键物理现象的建模能力。理解其局限，并与系统辨识、域适应等方法组合使用，是实际 Sim2Real 项目中的最佳实践。
