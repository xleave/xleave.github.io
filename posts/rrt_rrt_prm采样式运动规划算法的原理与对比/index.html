RRT / RRT* / PRM 采样式运动规划算法的原理与对比

基于采样的运动规划（Sampling-Based Motion Planning）是高维构型空间（Configuration Space）中求解机械臂无碰撞路径的主流方法。与基于搜索的方法（如 A*）不同，采样方法不需要对空间进行显式离散化，通过在连续空间中随机采样构建可行路径。

1. 构型空间与运动规划问题

1.1 构型空间

机械臂的构型空间（C-Space）是所有关节角度构成的空间。对于 nnn 自由度机械臂，构型空间为 Rn\mathbb{R}^nRn（或考虑关节限位的子空间）。

  • 自由空间 CfreeC_{\text{free}}Cfree​：机械臂不与任何障碍物碰撞的构型集合。
  • 障碍空间 CobsC_{\text{obs}}Cobs​：会导致碰撞的构型集合。

1.2 规划问题

给定起始构型 qstart∈Cfreeq_{\text{start}} \in C_{\text{free}}qstart​∈Cfree​ 和目标构型 qgoal∈Cfreeq_{\text{goal}} \in C_{\text{free}}qgoal​∈Cfree​，找到一条完全在 CfreeC_{\text{free}}Cfree​ 中的路径 σ:[0,1]→Cfree\sigma: [0, 1] \rightarrow C_{\text{free}}σ:[0,1]→Cfree​，连接起始和目标。

1.3 为什么采样方法？

7-DOF 机械臂的构型空间是 7 维的。基于网格的搜索方法在 7 维空间中的节点数呈 O(N7)O(N^7)O(N7) 增长——即使每个维度只取 100 个离散值，总节点数也达 101410^{14}1014，完全不可行。

采样方法通过在连续空间中随机采样有限数量的点来探索高维空间，避免了维度灾难。

2. RRT（Rapidly-exploring Random Tree）

2.1 核心思想

RRT（LaValle, 1998）从起点出发逐步生长一棵随机树，快速覆盖自由空间。

2.2 算法流程

function RRT(q_start, q_goal, max_iterations):
    T ← {q_start}   // 树的初始节点

    for i = 1 to max_iterations:
        q_rand ← random_sample(C)         // 随机采样
        q_near ← nearest(T, q_rand)        // 找树上最近节点
        q_new ← steer(q_near, q_rand, δ)   // 向采样点方向扩展 δ 步长
        if collision_free(q_near, q_new):   // 碰撞检测
            T.add_edge(q_near, q_new)
            if distance(q_new, q_goal) < ε:
                return extract_path(T, q_start, q_new)

    return failure

2.3 Voronoi 偏置

RRT 的随机采样天然具有 Voronoi 偏置特性：树的叶节点周围的 Voronoi 区域面积更大，因此更可能被采样到。这使得树倾向于向未探索区域扩展。

2.4 性能特性

指标值
概率完备性✓（若路径存在，随迭代次数趋于无穷，找到路径的概率趋于 1）
最优性✗（找到的路径通常非最优）
时间复杂度每步 O(nlog⁡n)O(n \log n)O(nlogn)（使用 KD-tree 加速最近邻查询）

3. RRT*（最优 RRT）

3.1 核心改进

RRT*（Karaman & Frazzoli, 2011）在 RRT 基础上引入两个关键操作，使算法在迭代次数趋于无穷时收敛到最优路径：

重新连接（Rewiring）：对于新添加的节点 qnewq_{\text{new}}qnew​，在其邻域内搜索所有节点，检查是否存在经过 qnewq_{\text{new}}qnew​ 的更短路径。如果存在，更新父节点。

function RRT_Star(q_start, q_goal, max_iterations):
    T ← {q_start}

    for i = 1 to max_iterations:
        q_rand ← random_sample(C)
        q_near ← nearest(T, q_rand)
        q_new ← steer(q_near, q_rand, δ)

        if collision_free(q_near, q_new):
            // 在邻域内找最优父节点
            Q_near ← near(T, q_new, radius)
            q_min ← q_near
            c_min ← cost(q_near) + dist(q_near, q_new)

            for q_n in Q_near:
                if collision_free(q_n, q_new):
                    c = cost(q_n) + dist(q_n, q_new)
                    if c < c_min:
                        q_min ← q_n
                        c_min ← c

            T.add_edge(q_min, q_new)

            // 重新连接：检查是否可以通过 q_new 改善其他节点
            for q_n in Q_near:
                if cost(q_new) + dist(q_new, q_n) < cost(q_n):
                    if collision_free(q_new, q_n):
                        T.rewire(parent(q_n) → q_new)

    return extract_best_path(T)

3.2 性能特性

指标值
概率完备性✓
渐进最优性✓（随迭代次数增加，路径代价收敛到最优）
收敛速度比 RRT 慢（额外的邻域搜索和重连操作）

4. PRM（Probabilistic Roadmap）

4.1 核心思想

PRM（Kavraki et al., 1996）采用预处理 + 查询的两阶段方法：

阶段 1：构建路线图（Preprocessing）

在自由空间中随机采样大量节点，对距离较近的节点对检查碰撞并添加边，构建一个图。

阶段 2：查询（Query）

将起点和终点连接到路线图，使用图搜索算法（如 A*）在路线图中找最短路径。

4.2 算法流程

// 预处理阶段
function Build_PRM(N, k):
    G ← empty graph
    for i = 1 to N:
        q ← random_sample(C_free)
        G.add_node(q)
    for each node q in G:
        neighbors ← k_nearest(G, q, k)
        for q_n in neighbors:
            if collision_free(q, q_n):
                G.add_edge(q, q_n, weight=dist(q, q_n))
    return G

// 查询阶段
function Query_PRM(G, q_start, q_goal):

    return A_Star(G, q_start, q_goal)

6. 工程优化技巧

6.1 目标偏置采样

以一定概率（如 5-10%）直接采样目标点而非随机点，加速树向目标方向生长：

if random() < goal_bias:
    q_rand = q_goal
else:
    q_rand = random_sample(C)

6.2 双向 RRT（Bi-directional RRT）

从起点和终点同时生长两棵树，尝试连接两棵树。通常比单向 RRT 快数倍。

6.3 碰撞检测加速

碰撞检测通常占运动规划总时间的 90% 以上。优化方法包括：

  • 使用简化碰撞几何体（球体/胶囊体包裹）。
  • 分层碰撞检测（先粗后细）。
  • 惰性碰撞检测（先规划再验证）。

6.4 路径平滑

采样规划产生的路径通常有锯齿状弯曲。常用平滑方法：

  • 快捷连接（Shortcutting）：随机选择路径上两点，检查直连是否无碰撞。
  • B-样条平滑：对关键路径点进行样条插值。

7. 选型建议

场景推荐算法原因
在线单次规划Bi-directional RRT快速找到可行路径
需要高质量路径RRT* + 路径平滑渐进最优
固定环境多次查询PRM预处理一次，查询极快
动态环境RRT / RRT*（增量更新）无需预处理
狭窄通道PRM 或 RRT 的桥接采样变体更好的覆盖性

总结

基于采样的运动规划通过随机采样避免了高维空间的维度灾难，是机械臂运动规划的标准方法。RRT 以快速探索见长，RRT* 在此基础上提供渐进最优性，PRM 通过预处理实现高效的多查询规划。工程实践中，双向 RRT + 路径平滑是最常用的组合，碰撞检测的优化则是提升整体性能的关键瓶颈。
