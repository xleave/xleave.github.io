模仿学习三范式对比：行为克隆、DAgger 与逆强化学习

模仿学习（Imitation Learning）从专家演示中学习策略，无需设计奖励函数。根据学习机制的不同，模仿学习可分为三种核心范式：行为克隆（Behavioral Cloning）、DAgger（Dataset Aggregation）和逆强化学习（Inverse Reinforcement Learning）。

1. 行为克隆（Behavioral Cloning, BC）

1.1 原理

将策略学习视为监督学习：给定专家演示数据集 D={(si,ai)}\mathcal{D} = \{(s_i, a_i)\}D={(si​,ai​)}，直接训练策略 πθ\pi_\thetaπθ​ 拟合状态-动作映射。

LBC=E(s,a)∼D[∥πθ(s)−a∥2]\mathcal{L}_{\text{BC}} = \mathbb{E}_{(s,a) \sim \mathcal{D}} \left[ \| \pi_\theta(s) - a \|^2 \right]LBC​=E(s,a)∼D​[∥πθ​(s)−a∥2]

1.2 优势与局限

优势局限
实现极简，标准监督学习分布偏移（Compounding Error）
不需要环境交互需要大量高质量演示
训练稳定、收敛快无法超越专家性能

1.3 分布偏移问题

BC 的核心问题是分布偏移（Distribution Shift）：

训练时，策略在专家访问过的状态分布 dπE(s)d_{\pi_E}(s)dπE​​(s) 上学习。部署时，策略的微小误差导致偏离专家轨迹，进入训练数据中未覆盖的状态。在这些新状态上，策略的预测不可靠，导致更大的偏离。误差像滚雪球一样累积。

对于时域长度为 TTT 的任务，BC 的误差上界为 O(T2)O(T^2)O(T2)——随时域长度二次增长。

2. DAgger（Dataset Aggregation）

2.1 原理

DAgger（Ross et al., 2011）通过迭代式数据收集解决分布偏移：

for i = 1, 2, ..., N:
    1. 使用当前策略 π_i 在环境中执行，收集轨迹
    2. 在策略访问的状态上，查询专家标注（正确动作）
    3. 将新数据加入训练集 D
    4. 在聚合数据集 D 上重新训练策略 π_{i+1}

2.2 核心洞察

DAgger 的关键在于：在策略自己会访问到的状态分布上获取专家指导。这确保策略在其实际执行时可能遇到的状态上都有训练数据。

理论上，DAgger 的误差上界为 O(T)O(T)O(T)——线性增长而非二次增长。

2.3 局限

局限说明
需要在线专家训练过程中需要反复查询专家，成本高
专家必须可交互不适用于只有离线演示的场景
安全问题策略执行阶段可能产生危险行为

3. 逆强化学习（Inverse Reinforcement Learning, IRL）

3.1 原理

IRL 不直接学习策略，而是从专家演示中推断奖励函数，然后使用标准 RL 在推断出的奖励下训练策略。

r∗=arg⁡max⁡r(EπE[∑tr(st,at)]−max⁡πEπ[∑tr(st,at)])r^* = \arg\max_r \left( \mathbb{E}_{\pi_E}[\sum_t r(s_t, a_t)] - \max_\pi \mathbb{E}_\pi[\sum_t r(s_t, a_t)] \right)r∗=argmaxr​(EπE​​[∑t​r(st​,at​)]−maxπ​Eπ​[∑t​r(st​,at​)])

直觉：找到一个奖励函数，使得专家策略在该奖励下是最优的，而其他策略表现更差。

3.2 对抗式 IRL（GAIL）

GAIL（Generative Adversarial Imitation Learning, Ho & Ermon, 2016）将 IRL 转化为对抗训练：

  • 判别器 DDD：区分专家轨迹和策略轨迹。
  • 策略 π\piπ：生成让判别器无法区分的轨迹。

min⁡πmax⁡DEπE[log⁡D(s,a)]+Eπ[log⁡(1−D(s,a))]\min_\pi \max_D \mathbb{E}_{\pi_E}[\log D(s, a)] + \mathbb{E}_\pi[\log(1 - D(s, a))]minπ​maxD​EπE​​[logD(s,a)]+Eπ​[log(1−D(s,a))]

判别器的输出可以视为隐式的奖励信号。

3.3 优势与局限

优势局限
可以超越专家性能需要环境交互（RL 训练）
学到的奖励可迁移到新环境训练不稳定（GAN 训练难度）
不受分布偏移影响计算成本高
处理多模态行为奖励函数不唯一

4. 三范式对比

维度行为克隆DAgger逆强化学习
学习目标直接学动作直接学动作先学奖励再学策略
环境交互不需要需要需要
专家类型离线演示在线可查询离线演示
分布偏移严重（O(T2)O(T^2)O(T2)）缓解（O(T)O(T)O(T)）不受影响
能否超越专家不能不能可以
实现复杂度低中高
训练稳定性高高低（GAN）
数据需求大量演示较少演示 + 迭代较少演示

5. 实践中的选择

场景推荐方法原因
有大量高质量演示，任务时域短行为克隆简单有效
可以在线查询专家DAgger解决分布偏移
只有少量演示，需要高性能IRL / GAIL利用 RL 探索
VLA 模型预训练行为克隆大规模数据 + 监督学习
需要奖励函数用于其他任务IRL奖励可迁移

6. 与 VLA 的关系

当前的 VLA 模型训练本质上是大规模行为克隆：

  • 使用大规模机器人操作数据集（如 Open X-Embodiment）。
  • 监督学习拟合观测-动作映射。
  • 通过数据量和模型容量缓解分布偏移。

但 VLA + 在线微调的组合实质上是 BC → DAgger 的思路：先用离线数据预训练，再在部署环境中少量迭代改进。

总结

行为克隆以极简的监督学习实现策略学习，但受限于分布偏移；DAgger 通过迭代式在线数据收集缓解偏移，但需要可交互的专家；逆强化学习从演示中推断奖励函数，能超越专家但训练复杂度高。三种范式各有适用场景，实践中常常组合使用——如以行为克隆预训练，再用在线 RL/DAgger 微调。
