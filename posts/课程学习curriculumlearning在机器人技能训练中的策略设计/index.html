课程学习（Curriculum Learning）在机器人技能训练中的策略设计

#课程学习#强化学习#机器人#技能训练#自适应

课程学习（Curriculum Learning）的核心思想是：按照从易到难的顺序呈现训练任务，使学习者逐步掌握复杂技能。在机器人 RL 训练中，直接面对最终任务的难度可能导致探索困难和训练不收敛。课程学习通过构建难度梯度，引导策略从简单子任务开始逐步提升能力。

1. 为什么机器人训练需要课程？

1.1 探索稀疏性

复杂操作任务中，成功条件要求多个子动作的精确协调。随机探索几乎不可能偶然触发成功。

1.2 奖励稀疏性

即使设计了密集奖励，初始随机策略可能完全无法获得有意义的奖励信号。

1.3 技能前置依赖

某些技能依赖于其他技能的掌握。例如，“在杯中倒水”要求先学会”稳定抓取杯子”。

2. 课程设计的维度

课程学习的关键是定义”难度”。在机器人任务中，常见的难度维度包括：

维度简单配置困难配置
目标距离目标紧邻末端执行器目标在工作空间远端
初始状态从接近目标的状态开始从随机初始状态开始
物体多样性单一固定物体多种形状、大小的物体
干扰因素无障碍物或干扰有障碍物和动态干扰
精度要求宽松的成功阈值严格的成功阈值
任务链长度单步子任务多步复合任务
物理参数固定参数域随机化范围扩大

3. 课程策略

3.1 手动课程

预先定义固定的难度序列和阶段转换条件：

stages = [
    {"target_distance": 0.05, "threshold": 0.9},  # 阶段 1
    {"target_distance": 0.15, "threshold": 0.8},  # 阶段 2
    {"target_distance": 0.30, "threshold": 0.7},  # 阶段 3
]

for stage in stages:
    while success_rate < stage["threshold"]:
        train(target_distance=stage["target_distance"])
    advance_to_next_stage()

优势：直觉简洁，可控性高。

局限：需要领域知识设计阶段，阶段数和转换条件需手动调优。

3.2 自动课程（Automatic Curriculum）

根据策略的当前能力自动调整任务难度。

基于成功率的自适应：

TAMER / Reverse Curriculum：从目标状态向前回溯——从距离目标最近的状态开始训练，逐步扩展到更远的初始状态。

3.3 Asymmetric Self-Play

两个智能体对抗：

  • Alice：设置越来越难的任务目标。
  • Bob：尝试完成 Alice 设置的任务。

Alice 的奖励是让 Bob 失败，Bob 的奖励是完成任务。均衡状态下，Alice 设置的任务难度恰好匹配 Bob 的能力边界。

3.4 基于学习进度的课程

选择策略正在快速进步的难度水平进行训练（Learning Progress）：

LP(d)=∣SRt(d)−SRt−k(d)∣\text{LP}(d) = |SR_t(d) - SR_{t-k}(d)|LP(d)=∣SRt​(d)−SRt−k​(d)∣

优先训练学习进度最高的难度级别——既不太简单（已掌握）也不太难（无法学习）。

4. 反向课程学习（Reverse Curriculum）

4.1 原理

从任务的目标状态出发，逐步扩展到更远的初始状态：

阶段 1: 距目标 1 步 → 策略学到最后一步
阶段 2: 距目标 2 步 → 策略学到最后两步
...
阶段 N: 从任意初始状态 → 策略掌握完整任务

4.2 优势

  • 在每个阶段，成功条件都很容易达到（只需少量步骤）。
  • 奖励信号密集——每个阶段的时域都很短。
  • 天然解决了奖励稀疏问题。

4.3 限制

需要能在仿真中重置到任意状态——真实环境中通常不可行。

5. 与域随机化的协同

课程学习可以与域随机化结合使用，形成”参数课程”：

阶段 1: 固定物理参数训练 → 策略学到基本行为
阶段 2: 小范围域随机化 → 策略获得初步鲁棒性
阶段 3: 逐步扩大随机化范围 → 策略达到 Sim2Real 所需的鲁棒性

这正是 OpenAI ADR（Automatic Domain Randomization）的核心思想。

6. 多任务课程

在多任务训练中，课程学习还涉及任务采样策略：

策略描述
均匀采样每个任务等概率采样
优先困难任务成功率低的任务采样概率更高
优先学习进度学习进度最高的任务优先
前置依赖排序先训练前置技能（如先抓取再放置）

7. 实践建议

场景推荐课程策略
已知任务的难度层次手动课程 + 成功率转换条件
难度维度复杂自动课程（基于成功率/学习进度）
稀疏奖励任务反向课程
Sim2Real 训练参数课程（与 ADR 协同）
多任务训练基于学习进度的任务采样

总结

课程学习通过构建难度梯度将复杂任务分解为渐进的学习阶段，有效缓解了 RL 训练中的探索困难和奖励稀疏问题。自动课程策略（如基于成功率的自适应和学习进度选择）减少了人工设计的负担。与域随机化的协同使用（ADR）是当前 Sim2Real 训练的最佳实践之一。
