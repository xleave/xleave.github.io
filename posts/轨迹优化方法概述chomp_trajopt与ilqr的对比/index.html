轨迹优化方法概述：CHOMP、TrajOpt 与 iLQR 的对比

轨迹优化（Trajectory Optimization）将运动规划问题建模为数学优化问题，通过迭代优化一条初始轨迹来满足动力学约束、避免碰撞并最小化某种代价函数。与基于采样的方法（如 RRT）不同，轨迹优化产生的轨迹通常更平滑、更高效。

1. 问题形式化

轨迹优化的一般形式为：

min⁡ξC(ξ)=∑t=0Tc(qt,q˙t)+cobs(qt)\min_{\xi} \quad C(\xi) = \sum_{t=0}^{T} c(q_t, \dot{q}_t) + c_{\text{obs}}(q_t)minξ​C(ξ)=∑t=0T​c(qt​,q˙​t​)+cobs​(qt​)

s.t.qt∈Cfree,q0=qstart,qT=qgoal\text{s.t.} \quad q_t \in C_{\text{free}}, \quad q_0 = q_{\text{start}}, \quad q_T = q_{\text{goal}}s.t.qt​∈Cfree​,q0​=qstart​,qT​=qgoal​

其中：

  • ξ=(q0,q1,…,qT)\xi = (q_0, q_1, \ldots, q_T)ξ=(q0​,q1​,…,qT​) 是离散化的轨迹。
  • c(qt,q˙t)c(q_t, \dot{q}_t)c(qt​,q˙​t​) 是运动代价（如关节速度、加速度的平方和）。
  • cobs(qt)c_{\text{obs}}(q_t)cobs​(qt​) 是障碍物代价。

2. CHOMP

2.1 核心思想

CHOMP（Covariant Hamiltonian Optimization for Motion Planning, Ratliff et al., 2009）使用协变梯度下降法直接优化轨迹。其特点是将碰撞代价定义为机械臂体积扫过的障碍物场积分。

2.2 目标函数

C(ξ)=Csmooth(ξ)+λCobs(ξ)C(\xi) = C_{\text{smooth}}(\xi) + \lambda C_{\text{obs}}(\xi)C(ξ)=Csmooth​(ξ)+λCobs​(ξ)

平滑代价：

Csmooth=12ξTKξC_{\text{smooth}} = \frac{1}{2} \xi^T K \xiCsmooth​=21​ξTKξ

其中 KKK 是有限差分矩阵（近似于加速度的平方和），惩罚轨迹的不平滑性。

障碍物代价：

Cobs=∑t∫bodycsdf(x(qt,u))∥Ju(qt)q˙t∥duC_{\text{obs}} = \sum_t \int_{\text{body}} c_{\text{sdf}}(x(q_t, u)) \|J_u(q_t)\dot{q}_t\| duCobs​=∑t​∫body​csdf​(x(qt​,u))∥Ju​(qt​)q˙​t​∥du

其中 csdfc_{\text{sdf}}csdf​ 是符号距离场（Signed Distance Field）的代价函数，积分遍历机械臂的体表面点。

2.3 协变梯度

CHOMP 的关键创新是使用 K−1K^{-1}K−1 作为度量矩阵进行梯度下降：

ξk+1=ξk−ηK−1∇C(ξk)\xi_{k+1} = \xi_k - \eta K^{-1} \nabla C(\xi_k)ξk+1​=ξk​−ηK−1∇C(ξk​)

这使得梯度更新保持轨迹的平滑性——更新本身也是平滑的。

2.4 特点

优势局限
梯度更新保持平滑性可能陷入局部最优
不需要初始可行轨迹障碍物代价需要预计算 SDF
计算效率较高对狭窄间隙的规划困难

3. TrajOpt

3.1 核心思想

TrajOpt（Schulman et al., 2014）将碰撞避免建模为不等式约束（而非代价函数），使用序列凸优化（Sequential Convex Programming, SCP）求解。

3.2 优化问题

min⁡ξCsmooth(ξ)\min_\xi \quad C_{\text{smooth}}(\xi)minξ​Csmooth​(ξ)

s.t.dmin(qt)≥dsafe,∀t\text{s.t.} \quad d_{\text{min}}(q_t) \geq d_{\text{safe}}, \quad \forall ts.t.dmin​(qt​)≥dsafe​,∀t

其中 dmin(qt)d_{\text{min}}(q_t)dmin​(qt​) 是构型 qtq_tqt​ 下机械臂与障碍物的最小距离，dsafed_{\text{safe}}dsafe​ 是安全裕度。

3.3 序列凸优化

碰撞约束是非凸的。TrajOpt 在每次迭代中将其线性化：

dmin(qt)≈dmin(qtk)+∇dminT(qt−qtk)≥dsafed_{\text{min}}(q_t) \approx d_{\text{min}}(q_t^k) + \nabla d_{\text{min}}^T (q_t - q_t^k) \geq d_{\text{safe}}dmin​(qt​)≈dmin​(qtk​)+∇dminT​(qt​−qtk​)≥dsafe​

然后在信赖域内求解凸近似问题。信赖域随迭代自适应调整。

3.4 连续碰撞检测

TrajOpt 引入连续碰撞检测（Continuous Collision Detection），检查相邻时间步之间的扫掠体是否与障碍物碰撞，避免由于时间步过大而”穿过”薄障碍物。

3.5 特点

优势局限
碰撞作为硬约束（安全保证更强）需要高效的碰撞距离查询
收敛快（信赖域 + 凸优化）可能陷入局部最优
支持连续碰撞检测SCP 的线性化可能不精确

4. iLQR（iterative Linear Quadratic Regulator）

4.1 核心思想

iLQR（也称 DDP, Differential Dynamic Programming 的简化版本）是最优控制领域的方法，通过在当前轨迹附近二次近似代价函数和线性化动力学来迭代优化轨迹。

4.2 与 CHOMP/TrajOpt 的区别

iLQR 显式地考虑动力学约束，适用于需要物理可行轨迹的场景（如飞行器、腿式机器人）。CHOMP 和 TrajOpt 通常假设运动学模型，不考虑力矩限制等动力学约束。

4.3 算法流程

1. 初始化轨迹 (q_0:T, u_0:T-1)
2. 前向模拟：按当前控制序列模拟轨迹
3. 反向传播：沿轨迹反向计算最优控制修正
   - 线性化动力学：q_{t+1} ≈ A_t q_t + B_t u_t
   - 二次近似代价：C ≈ 1/2 δx^T Q δx + δu^T R δu
   - 求解 Riccati 方程得到最优反馈增益 K_t
4. 前向更新：应用修正后的控制序列
5. 线搜索：选择合适的步长
6. 重复 2-5 直到收敛

4.4 特点

优势局限
考虑动力学约束需要动力学模型
产生力矩可行的轨迹对初始轨迹敏感
计算效率高（线性复杂度 in T）不等式约束处理不自然
广泛用于 MPC碰撞避免需要额外处理

6. 与采样规划的互补使用

轨迹优化的局部最优问题可以通过与采样规划结合来缓解：

  1. RRT/PRM 提供初始轨迹：采样方法找到粗略可行路径。
  2. 轨迹优化精修：CHOMP/TrajOpt 将粗略路径优化为平滑、高效的轨迹。

这种”采样 + 优化”的两阶段方法结合了采样方法的全局探索能力和轨迹优化的局部精修能力。

7. 选型建议

场景推荐方法原因
机械臂运动学规划TrajOpt硬碰撞约束 + 快速收敛
需要极平滑轨迹CHOMP协变梯度保持平滑性
有动力学约束（MPC）iLQR原生支持动力学
复杂障碍物环境RRT* → TrajOpt全局探索 + 局部精修
实时重规划iLQR (MPC 模式)计算效率高

总结

CHOMP、TrajOpt 和 iLQR 代表了轨迹优化的三种主流范式。CHOMP 以协变梯度和 SDF 碰撞场见长，TrajOpt 以硬碰撞约束和序列凸优化见长，iLQR 以动力学约束和最优控制理论见长。三者都面临局部最优问题，与采样规划方法的结合使用是工程实践中的最佳方式。
