博客配色系统的设计思路：基于 HSL 的动态主题

#CSS#设计#主题#Material Design

背景

大多数博客主题提供的颜色切换只是在几套预设配色之间进行切换。而本站采用的是更为灵活的方案：用户可以自由调整一个色相（Hue）数值，整套配色系统随之实时更新，无需预设任何调色板。

这一方案的核心思路来源于 Material Design 3（MD3）的动态配色（Dynamic Color）规范：基于单一主色，通过固定的色相偏移和饱和度/明度调节，生成语义化的一套颜色 token。

HSL 颜色空间的优势

相比 HEX 或 RGB，HSL（Hue 色相 / Saturation 饱和度 / Lightness 明度）更接近人类的感知方式，且非常适合程序化的配色计算：

  • 只改变色相值（H），保持 S 和 L 不变，可以轻松地在不同色系之间切换，且颜色整体亮度与饱和度保持一致。
  • 调节 L 值可以生成同一色系的浅色、中色、深色变体，适合 container / on-container 的配色关系。
  • CSS hsl() 原生支持，无需 JavaScript 计算，可以在 CSS 变量中直接引用。

颜色 Token 结构

本站的颜色系统由以下几类 CSS 自定义变量构成：

/* 以 --accent-hue 为核心色相 */
:root {
  --accent-hue: 255; /* 用户可调节，范围 0-360 */
}

[data-theme="light"] {
  /* 主色系 */
  --md-primary:            hsl(var(--accent-hue), 65%, 45%);
  --md-on-primary:         hsl(var(--accent-hue), 65%, 98%);
  --md-primary-container:  hsl(var(--accent-hue), 65%, 90%);
  --md-on-primary-container: hsl(var(--accent-hue), 65%, 20%);

  /* 次色系（色相偏移 +60°）*/
  --md-secondary:          hsl(calc(var(--accent-hue) + 60), 40%, 45%);
  
  /* 中性色（极低饱和度，保留色调倾向）*/
  --md-surface:            hsl(var(--accent-hue), 8%, 96%);
  --md-surface-card:       hsl(var(--accent-hue), 8%, 99%);
  --md-on-surface:         hsl(var(--accent-hue), 8%, 10%);
}

暗色模式

暗色模式下，主色需要适当提亮（因为暗背景上的颜色感知亮度会降低），容器色则改为深色调：

[data-theme="dark"] {
  --md-primary:            hsl(var(--accent-hue), 65%, 70%); /* 提亮 */
  --md-primary-container:  hsl(var(--accent-hue), 45%, 25%); /* 深色容器 */
  --md-on-primary-container: hsl(var(--accent-hue), 65%, 85%);

  --md-surface:            hsl(var(--accent-hue), 8%, 8%);
  --md-surface-card:       hsl(var(--accent-hue), 8%, 12%);
  --md-on-surface:         hsl(var(--accent-hue), 8%, 92%);
}

OLED 黑色模式

OLED 屏幕下真正的纯黑（#000000）像素不发光，有助于节省电量。OLED 模式在暗色基础上进一步将背景设为纯黑：

[data-theme="oled"] {
  --md-surface:       hsl(0, 0%, 0%);
  --md-surface-card:  hsl(var(--accent-hue), 5%, 5%);
}

色相调节的实现

用户通过侧边栏的色相滑块更改颜色：

HTML：

<input type="range" id="hue-slider" min="0" max="360" value="255" />

JavaScript：

const slider = document.getElementById('hue-slider');

slider.addEventListener('input', () => {
  const hue = slider.value;
  document.documentElement.style.setProperty('--accent-hue', hue);
  localStorage.setItem('accent-hue', hue);
});

// 页面加载时恢复用户设置
const savedHue = localStorage.getItem('accent-hue') || '255';
document.documentElement.style.setProperty('--accent-hue', savedHue);
slider.value = savedHue;

关键在于将 --accent-hue 设置在 :root（即 document.documentElement）上，从而让所有使用该变量的子元素都能自动响应更新。

颜色对比度与可读性

动态配色的风险在于：用户选择的色相可能导致某些颜色组合的对比度不足（尤其是中明度色相如黄绿色系）。

处理策略：

  1. 控制 primary 的明度范围：亮色模式中，primary 固定在 35%-50% 明度区间；暗色模式固定在 65%-80% 区间，避免极端值。

  2. on-primary 始终使用极端明度：on-primary（主色上方的文字色）在亮色模式下固定为接近白色，在暗色模式下固定为接近黑色，确保对比度始终达标。

  3. 中性色保留色调倾向：表面色（surface、on-surface）使用极低饱和度（5%-10%）而非纯灰色（0%），使整体配色具有”同色调的灰”的视觉和谐感，这是 MD3 配色规范中的重要细节。

主题切换的无闪烁实现

如果主题信息（如 data-theme 属性）在 JavaScript 加载完成后才被应用，用户会在页面加载瞬间看到默认颜色的”闪白”现象（FOUC，Flash of Unstyled Content）。

解决方案：在 <head> 中内联一段极小的同步脚本，在 HTML 解析时立即应用主题：

<head>
  <script>
    // 同步执行，无需等待 DOM Ready
    const theme = localStorage.getItem('theme') || 'dark';
    const hue = localStorage.getItem('accent-hue') || '255';
    document.documentElement.setAttribute('data-theme', theme);
    document.documentElement.style.setProperty('--accent-hue', hue);
  </script>
</head>

由于该脚本在 CSS 文件加载之前执行，且 data-theme 属性已设置好，CSS 中对应的变量会正确应用，不会产生闪屏。

小结

本站的动态配色系统由以下几个核心机制构成：

机制说明
单一色相变量--accent-hue 驱动所有派生色
HSL 语义化推导固定 S/L 配置生成各功能色
CSS 变量层叠data-theme 属性切换不同主题的变量集
localStorage 持久化跨页面、跨会话保持用户设置
内联同步脚本避免主题切换的 FOUC 问题

这套方案无需引入任何 CSS-in-JS 库或主题框架，纯粹依靠 CSS 自定义变量即可实现，非常适合轻量级静态站点使用。
